# 腾讯云Agent Memory：Mermaid无限画布×上下文卸载

> TencentDB Agent Memory 通过上下文卸载 + Mermaid 无限画布，把长任务过程转化为结构化记忆。超长 Session 实验中最高节省 61% Token，任务通过率最多提升 52%。

## 文章核心脉络

文章从"压缩的艺术"出发，层层递进：

1. **为什么压缩？** → 长任务中上下文被过程信息撑爆，问题不是信息太多而是没有被"压缩成最可用的形式"
2. **怎么压缩？** → 符号化压缩（三条原则：通用知识、生成不复杂、表达自由）→ Mermaid 作为最优载体
3. **压缩后如何组织？** → 从线性摘要到 Mermaid 无限画布（任务拓扑图），Flowchart > StateDiagram ~15%
4. **工程实践？** → 四级折叠（Raw→JSONL→MMD→Metadata）+ 逐级找回
5. **实验验证？** → 四组超长 Session 压力测试 + 消融实验

## 提取的关键概念卡片

| 概念 | 核心洞察 | 详见 |
|------|---------|------|
| **语言压缩哲学** | 压缩依赖语义结构保留而非字面保留——"一语胜千言"不是包含全部细节而是抓住代表整体的结构 | [[语言压缩哲学]] |
| **符号化压缩** | 不是让模型"记住什么符号"而是让模型"能从符号中推理出什么结构"；三条原则：通用知识/生成不复杂/表达自由 | [[符号化压缩]] |
| **上下文卸载** | 不是所有信息都要一直放在模型眼前——暂时不需要的搬到外部，只保留摘要和索引 | [[上下文卸载]] |
| **Mermaid任务画布** | 把离散摘要组织成任务拓扑图——线性列表像日志，图结构像地图 | [[Mermaid任务画布]] |
| **层次化注意力** | Overview→Focus→Detail 三层导航，让注意力变有层次而非扁平 | [[层次化注意力机制]] |
| **四级折叠架构** | Raw→JSONL→MMD→Metadata 逐级变轻，逐级可找回 | [[TencentDB Agent Memory]] |

## 查漏补缺要点（补充到已有笔记的新增视角）

### 对 [[记忆系统]] 的补充
- 短期记忆的"管理与压缩"有了新的生产化实践——四级折叠 + Mermaid 画布
- 市场方案对比新增：TencentDB Agent Memory（四级折叠架构，文件系统依赖）
- 长期个性化记忆的评测数据：PersonaMem 准确率相对提升 59%

### 对 [[上下文压缩]] 的补充
- 新增第三种策略维度：**卸载 vs 压缩**——卸载是"搬到档案柜"，压缩是"缩成一句话"
- 消融实验证明：单独卸载效果有限（~15% token节省），卸载+结构化画布效果翻倍（31-33%）
- 符号化压缩三原则为压缩载体选择提供了设计框架

### 对 [[可逆压缩]] 的补充
- 四级折叠架构天然支持可逆性——每一层都有索引指向下一层，原文在外部文件中
- 与 OpenCode 的"时间戳标记"不同：TencentDB 用结构化索引（node_id → JSONL → result_ref → refs）而非简单占位符
- 实现方式：全文依赖文件系统，而非数据库标记

### 对 [[上下文腐蚀与注意力衰减]] 的补充
- Anthropic 实验数据：上下文超过 max window 80% 时发生"上下文腐烂"
- 层次化注意力是对抗腐蚀的具体方案——不让所有信息竞争注意力，而是分层导航
- 新增三种应对策略：上下文卸载、符号化压缩（Mermaid画布）、层次化注意力机制

### 对 [[四级水位线方案]] 的补充
- MUR AI 的四级水位线(Tier 0-3)和 TencentDB 的四级折叠(Level 0-3)是两种不同的"分层"哲学：
  - MUR AI：同一信息的不同压缩强度（Tier是强度递增）
  - TencentDB：同一信息在不同粒度的表示（Level是粒度递减）

### 对 [[智能体评估]] 的补充
- 新增"超长Session串行评测方法论"：把50+任务串在同一session中连续执行
- 比单题独立评测更接近真实Agent使用场景，但绝对分数更低
- 评估重点在相对变化而非绝对分数
