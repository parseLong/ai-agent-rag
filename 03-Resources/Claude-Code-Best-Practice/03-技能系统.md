# Claude Code 技能系统 (Skills)

## 什么是 Skill？

Skill 是可复用的能力模块，存储在 `.claude/skills/<name>/SKILL.md`。有两种使用模式：

1. **Agent Skill (Preloaded)** — 通过 `skills:` 字段预加载到 Agent 上下文中
2. **Skill (Direct Invocation)** — 通过 `Skill` 工具直接调用

## Frontmatter 字段（15个）

| 字段 | 类型 | 必需 | 说明 |
|------|------|:---:|------|
| `name` | string | 否 | 显示名称和 `/slash-command` 标识符。默认使用目录名 |
| `description` | string | 推荐 | 技能描述。用于自动补全和自动发现 |
| `when_to_use` | string | 否 | 调用场景补充说明，追加到 description 中 |
| `argument-hint` | string | 否 | 自动补全提示（如 `[issue-number]`） |
| `arguments` | string/list | 否 | 命名位置参数，用于 `$name` 替换 |
| `disable-model-invocation` | boolean | 否 | 设为 `true` 阻止自动调用 |
| `user-invocable` | boolean | 否 | 设为 `false` 隐藏 `/` 菜单（仅作为背景知识） |
| `allowed-tools` | string | 否 | 技能激活时无需权限提示的工具 |
| `model` | string | 否 | 技能运行时的模型 |
| `effort` | string | 否 | 覆盖努力级别（low/medium/high/xhigh/max） |
| `context` | string | 否 | 设为 `fork` 在隔离子代理上下文中运行 |
| `agent` | string | 否 | `context: fork` 时的子代理类型（默认：`general-purpose`） |
| `hooks` | object | 否 | 该技能的生命周期钩子 |
| `paths` | string/list | 否 | 限制自动激活的文件 glob 模式 |
| `shell` | string | 否 | `` !`command` `` 块的 shell（`bash` 或 `powershell`） |

## 官方捆绑 Skills（6个）

| # | Skill | 说明 |
|---|-------|------|
| 1 | `simplify` | 审查代码复用、质量和效率，消除重复 |
| 2 | `batch` | 批量跨文件运行命令 |
| 3 | `debug` | 调试失败命令或代码问题 |
| 4 | `loop` | 按间隔循环运行提示或命令（最长3天） |
| 5 | `claude-api` | 使用 Claude API 或 Anthropic SDK 构建应用 |
| 6 | `fewer-permission-prompts` | 扫描常用命令并添加 allowlist 减少权限提示 |

## 两种 Skill 模式对比

### Agent Skill (Preloaded)

```yaml
# .claude/agents/weather-agent.md
---
name: weather-agent
skills:
  - weather-fetcher    # 预加载到 agent 上下文
---
```

- **注入方式**: 完整技能内容在启动时注入到 agent 上下文
- **使用方式**: Agent 遵循预加载技能的指令
- **无动态调用**: 技能是参考材料，不是单独调用的

### Skill (Direct Invocation)

```yaml
# .claude/skills/weather-svg-creator/SKILL.md
---
name: weather-svg-creator
description: Creates an SVG weather card...
---
```

- **调用方式**: `Skill(skill: "weather-svg-creator")`
- **独立执行**: 在命令的上下文中运行，不在 agent 内部
- **接收上下文数据**: 使用对话中已有的数据

## 示例：完整的 Skill 定义

```markdown
---
name: weather-fetcher
description: Fetch weather data from Open-Meteo API
argument-hint: '[city]'
allowed-tools: WebFetch
model: haiku
---

# Weather Fetcher

Fetch the current temperature for the given city using the Open-Meteo API.

## Usage

1. Accept the city name from the user
2. Call the Open-Meteo geocoding API to get coordinates
3. Call the Open-Meteo weather API for current temperature
4. Return the temperature and unit
```
