# Claude Code 编排工作流

## 架构概述

**Command → Agent → Skill** 是 Claude Code 的核心编排模式，将复杂任务分解为三个层次：

| 组件 | 角色 | 示例 |
|------|------|------|
| **Command** | 入口点、用户交互 | `/weather-orchestrator` |
| **Agent** | 数据获取（预加载 Skill） | `weather-agent` + `weather-fetcher` |
| **Skill** | 输出创建（独立调用） | `weather-svg-creator` |

## 执行流程

```
╔══════════════════════════════════════════════════════════════════╗
║              ORCHESTRATION WORKFLOW                              ║
║           Command  →  Agent  →  Skill                            ║
╚══════════════════════════════════════════════════════════════════╝

                         ┌───────────────────┐
                         │  User Interaction │
                         └─────────┬─────────┘
                                   │
                                   ▼
         ┌─────────────────────────────────────────────────────┐
         │  /weather-orchestrator — Command (Entry Point)      │
         └─────────────────────────┬───────────────────────────┘
                                   │
                              Step 1
                                   │
                                   ▼
                      ┌────────────────────────┐
                      │  AskUser — C° or F°?   │
                      └────────────┬───────────┘
                                   │
                         Step 2 — Agent tool
                                   │
                                   ▼
         ┌─────────────────────────────────────────────────────┐
         │  weather-agent — Agent ● skill: weather-fetcher     │
         └─────────────────────────┬───────────────────────────┘

                                   │
                         Step 3 — Skill tool
```

## 组件详解

### 1. Command（命令）

- **位置**: `.claude/commands/weather-orchestrator.md`
- **用途**: 入口点 — 编排工作流和处理用户交互
- **动作**:
  1. 询问用户温度单位偏好（摄氏度/华氏度）
  2. 通过 Agent 工具调用 weather-agent
  3. 通过 Skill 工具调用 weather-svg-creator

### 2. Agent + Preloaded Skill（代理 + 预加载技能）

- **位置**: `.claude/agents/weather-agent.md`
- **用途**: 使用预加载技能获取天气数据
- **技能**: `weather-fetcher`（作为领域知识预加载）
- **可用工具**: Read, Skill
- **模型**: sonnet

### 3. Skill（技能 — 独立调用）

- **位置**: `.claude/skills/weather-svg-creator/SKILL.md`
- **用途**: 创建 SVG 天气卡片并写入输出文件
- **调用方式**: 通过 Skill 工具从命令调用
- **输出**:
  - `weather.svg` — SVG 天气卡片
  - `output.md` — 天气摘要

## 关键设计原则

1. **两种技能模式**: 同时展示预加载的 agent skills 和直接调用的 skills
2. **命令作为编排器**: 命令处理用户交互并协调工作流
3. **代理用于数据获取**: 代理使用预加载技能获取数据，然后返回
4. **技能用于输出**: SVG 创建器独立运行，从命令上下文接收数据
5. **清晰分离**: 获取（代理）→ 渲染（技能）— 每个组件单一职责

## Agent Skill 配置示例

```yaml
# .claude/agents/weather-agent.md
---
name: weather-agent
description: Fetch weather data using preloaded skill
skills:
  - weather-fetcher
model: sonnet
color: green
memory: project
---
```

## 完整示例执行

```
Input: /weather-orchestrator
├─ Step 1: Asks: Celsius or Fahrenheit?
│  └─ User: Celsius
├─ Step 2: Agent tool → weather-agent
│  ├─ Preloaded Skill:
│  │  └─ weather-fetcher (domain knowledge)
│  ├─ Fetches from Open-Meteo → 26°C
│  └─ Returns: temperature=26, unit=Celsius
├─ Step 3: Skill tool → /weather-svg-creator
│  ├─ Creates: orchestration-workflow/weather.svg
│  └─ Writes: orchestration-workflow/output.md
└─ Output:
   ├─ Unit: Celsius
   ├─ Temperature: 26°C
   ├─ SVG: orchestration-workflow/weather.svg
   └─ Summary: orchestration-workflow/output.md
```
