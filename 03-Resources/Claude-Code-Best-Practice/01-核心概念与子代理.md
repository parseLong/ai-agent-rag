# 核心概念与子代理 (Subagents)

## 什么是 Subagent？

Subagent 是 Claude Code 中用于处理**复杂多步任务**的专用代理。与主会话不同，Subagent 可以：
- 在隔离上下文中运行任务
- 使用特定的模型和工具集
- 预加载特定技能
- 在后台运行

## Frontmatter 字段（16个）

| 字段                | 类型          |  必需   | 说明                                                                       |
| ----------------- | ----------- | :---: | ------------------------------------------------------------------------ |
| `name`            | string      | **是** | 唯一标识符，使用小写字母和连字符                                                         |
| `description`     | string      | **是** | 何时调用。用 `"PROACTIVELY"` 让 Claude 自动调用                                     |
| `tools`           | string/list |   否   | 允许的工具列表（如 `Read, Write, Edit, Bash`）。省略则继承所有工具                           |
| `disallowedTools` | string/list |   否   | 要禁用的工具                                                                   |
| `model`           | string      |   否   | 模型别名：`sonnet`、`opus`、`haiku`、完整模型 ID 或 `inherit`（默认）                     |
| `permissionMode`  | string      |   否   | 权限模式：`default`、`acceptEdits`、`auto`、`dontAsk`、`bypassPermissions`、`plan` |
| `maxTurns`        | integer     |   否   | 子代理停止前的最大自主轮数                                                            |
| `skills`          | list        |   否   | 预加载到代理上下文中的技能名称                                                          |
| `mcpServers`      | list        |   否   | 该子代理专用的 MCP 服务器                                                          |
| `hooks`           | object      |   否   | 该子代理的生命周期钩子                                                              |
| `memory`          | string      |   否   | 持久化记忆范围：`user`、`project` 或 `local`                                       |
| `background`      | boolean     |   否   | 设为 `true` 则始终以后台任务运行                                                     |
| `effort`          | string      |   否   | 覆盖努力级别：`low`、`medium`、`high`、`xhigh`、`max`                               |
| `isolation`       | string      |   否   | 设为 `"worktree"` 则在临时 git 工作树中运行                                          |
| `initialPrompt`   | string      |   否   | 作为子代理第一个用户提示自动提交                                                         |
| `color`           | string      |   否   | CLI 输出颜色（用于视觉区分）                                                         |

## 官方内置 Agent（5个）

| # | Agent | 模型 | 工具 | 用途 |
|---|-------|------|------|------|
| 1 | `general-purpose` | inherit | 全部 | 复杂多步任务 — 研究、代码搜索和自主工作的默认代理 |
| 2 | `Explore` | haiku | 只读（无 Write, Edit） | 快速代码库搜索和探索 |
| 3 | `Plan` | inherit | 只读（无 Write, Edit） | 计划模式中的预规划研究 |
| 4 | `statusline-setup` | sonnet | Read, Edit | 配置用户的 Claude Code 状态行 |
| 5 | `claude-code-guide` | haiku | Glob, Grep, Read, WebFetch, WebSearch | 回答 Claude Code 功能、Agent SDK 和 Claude API 的问题 |

## 使用示例

```yaml
# .claude/agents/weather-agent.md
---
name: weather-agent
description: PROACTIVELY fetch weather data
skills:
  - weather-fetcher
model: sonnet
color: green
memory: project
---
```

## 关键原则

- Subagent **不能**通过 bash 命令调用其他 Subagent — 必须使用 `Agent` 工具
- 在 Subagent 定义中明确说明工具使用方式
- 避免使用 "launch" 等模糊词汇，可能被误解为 bash 命令

## Subagent Orchestration 示例

```python
Agent(
    subagent_type="agent-name",
    description="...",
    prompt="...",
    model="haiku"
)
```
