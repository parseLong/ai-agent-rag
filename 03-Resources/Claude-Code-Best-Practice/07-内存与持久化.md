# Claude Code 内存与持久化

## CLAUDE.md 加载机制

### 两种加载机制

#### 1. 祖先加载（向上遍历目录树）

Claude Code 从当前工作目录**向上**遍历到文件系统根目录，加载找到的每个 CLAUDE.md。这些文件在启动时**立即加载**。

#### 2. 后代加载（向下遍历目录树）

子目录中的 CLAUDE.md 文件**不会在启动时加载**。它们只在会话期间读取到该子目录中的文件时才被包含。这称为**懒加载**。

### Monorepo 示例

```
/mymonorepo/
├── CLAUDE.md          # 根级指令（跨所有组件共享）
├── frontend/
│   └── CLAUDE.md      # 前端专用指令
├── backend/
│   └── CLAUDE.md      # 后端专用指令
└── api/
    └── CLAUDE.md      # API专用指令
```

### 从根目录启动

```bash
cd /mymonorepo
claude
```

| 文件 | 启动时加载? | 原因 |
|------|------------|------|
| `/mymonorepo/CLAUDE.md` | ✅ 是 | 当前工作目录 |
| `/mymonorepo/frontend/CLAUDE.md` | ❌ 否 | 仅在读取 frontend/ 文件时加载 |
| `/mymonorepo/backend/CLAUDE.md` | ❌ 否 | 仅在读取 backend/ 文件时加载 |
| `/mymonorepo/api/CLAUDE.md` | ❌ 否 | 仅在读取 api/ 文件时加载 |

### 从组件目录启动

```bash
cd /mymonorepo/frontend
claude
```

| 文件 | 启动时加载? | 原因 |
|------|------------|------|
| `/mymonorepo/CLAUDE.md` | ✅ 是 | 祖先目录 |
| `/mymonorepo/frontend/CLAUDE.md` | ✅ 是 | 当前工作目录 |
| `/mymonorepo/backend/CLAUDE.md` | ❌ 否 | 目录树的不同分支 |
| `/mymonorepo/api/CLAUDE.md` | ❌ 否 | 目录树的不同分支 |

## 关键要点

1. **祖先始终在启动时加载** — 确保始终可访问根级、仓库范围的指令
2. **后代懒加载** — 子目录 CLAUDE.md 仅在交互时加载
3. **兄弟目录不加载** — 在 frontend/ 工作时不会加载 backend/ 的 CLAUDE.md
4. **全局 CLAUDE.md** — 可放在 `~/.claude/CLAUDE.md`，应用于所有会话

## 最佳实践

1. **将共享约定放在根级 CLAUDE.md** — 编码标准、提交格式、PR 模板
2. **将组件专用指令放在组件 CLAUDE.md** — 框架特定模式、组件架构
3. **使用 CLAUDE.local.md 作为个人偏好** — 添加到 `.gitignore`
4. **保持 CLAUDE.md 在 200 行以内** — 确保可靠遵循

## Memory 存储位置

| 类型 | 位置 | 范围 |
|------|------|------|
| **Project** | `CLAUDE.md`, `.claude/rules/` | 项目级别 |
| **User** | `~/.claude/rules/`, `~/.claude/projects/<project>/memory/` | 用户级别 |
| **Global** | `~/.claude/CLAUDE.md` | 所有会话 |

## 记忆文件结构

```
~/.claude/
├── CLAUDE.md                 # 全局默认
├── rules/                    # 全局规则
└── projects/
    └── <project>/
        └── memory/           # 项目级记忆
            ├── user_role.md
            ├── feedback_testing.md
            └── project_deadline.md
```
