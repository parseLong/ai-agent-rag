## 跨模型工作流

使用 Claude Code 与其他模型（Codex、Gemini、GPT、Kimi、DeepSeek）协同的三种机制：

| 机制 | 说明 | 示例 |
|------|------|------|
| **Plugin** | 另一个模型的 CLI 在 Claude Code 内部运行 | `/codex:review` |
| **MCP** | Claude Code 通过 MCP 调用另一个模型 | 多模型 MCP 服务器 |
| **Router** | 替换 Claude Code 的 API 端点到不同提供商 | OpenRouter 等 |

### 跨模型工具

| 工具 | 类型 | 桥接到 | 说明 |
|------|------|--------|------|
| [claude-code-router](https://github.com/musistudio/claude-code-router) | Router | OpenRouter, DeepSeek, Ollama, Gemini 等 | 路由 API 到任何兼容提供商 |
| [CLIProxyAPI](https://github.com/router-for-me/CLIProxyAPI) | Router | Gemini CLI, Codex, Claude Code | 包装 CLI 为兼容 API |
| [codex-plugin-cc](https://github.com/openai/codex-plugin-cc) | Plugin | Codex / GPT-5 | 官方 OpenAI 插件 |
| [pal-mcp-server](https://github.com/BeehiveInnovations/pal-mcp-server) | MCP | Gemini, OpenAI, Azure 等 50+ 模型 | 多模型 MCP 服务器 |

## 推荐工作流模式

### 简单项目

```
/init → /plan → [开发] → /review → [提交]
```

### 中等复杂度

```
/init → /plan → /tdd → [开发 + 测试] → /review → /security-review → [提交 + PR]
```

### 复杂项目

```
/init → /ultraplan → /spec → /tdd → [开发 + 测试] → /review → /ultrareview → [合并 + 部署]
```
