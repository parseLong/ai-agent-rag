# 从0到1搭建Agent：原理分析及个人助手实践

> 作者：占旭鹏（阿里），2026年5月21日。本文约50分钟阅读时间，分为理论篇和实践篇两大部分。

## 📌 文章核心框架

文章按时间线梳理了Agent技术从LLM到完整系统的演进，本质是在解决三个递进问题：

1. **LLM知道什么**（知识 → 记忆 + RAG 扩展）
2. **LLM能做什么**（能力 → Function Call + MCP + Skill）
3. **LLM怎么做得好**（质量 → Agent Loop + Multi-Agent + Harness）

## 📝 个人笔记

这篇文章是近期看到的**最系统的Agent技术全景梳理**之一。它的价值在于：

1. **时间线叙事**：不是零散的概念罗列，而是"问题→催生方案→新问题→新方案"的递进逻辑
2. **实践与理论结合**：每个理论概念都有对应的代码实现和设计决策
3. **反直觉洞察**：如"几千字符的有界文件记忆比向量检索更实用"这类经验总结

最有启发性的三个观点：
- **记忆到技能转化**：从被动回忆到主动执行的递进，这是Agent真正"越用越懂你"的机制
- **Plan即工具**：不是ReAct和Plan两种模式切换，而是在ReAct循环中嵌入todo工具，零额外开销
- **渐进式加载**：10x Token效率的Skill advertise方案，解决了工具数量增长的上下文膨胀问题
