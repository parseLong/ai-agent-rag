# 智能旅行助手项目

## 🎯 项目目标

基于 Hello-Agents 第一章的代码，实现一个能够查询天气并推荐景点的智能旅行助手。

## 📋 项目背景

这是学习智能体的第一个实践项目，通过这个项目理解 `Thought-Action-Observation` 范式的实际运作。

## 🔧 技术栈

- **核心框架**：Python + OpenAI API
- **工具/库**：requests, tavily-python, openai
- **模型**：GPT-4 / 兼容 OpenAI 接口的 LLM

## 📝 实现步骤

### 步骤 1：环境准备
- [x] 安装依赖库
- [x] 获取 API 密钥

### 步骤 2：实现工具函数
- [x] `get_weather(city)` - 查询天气
- [x] `get_attraction(city, weather)` - 推荐景点

### 步骤 3：构建 Agent 循环
- [x] 设计 System Prompt
- [x] 实现 Thought-Action 解析
- [x] 实现 Observation 反馈

## 💻 核心代码

```python
# 关键代码实现
AGENT_SYSTEM_PROMPT = """
你是一个智能旅行助手。你的任务是分析用户的请求，并使用可用工具一步步地解决问题。

# 可用工具:
- `get_weather(city: str)`: 查询指定城市的实时天气。
- `get_attraction(city: str, weather: str)`: 根据城市和天气搜索推荐的旅游景点。

# 输出格式要求:
你的每次回复必须严格遵循以下格式：
Thought: [你的思考过程和下一步计划]
Action: [你要执行的具体行动]
"""
```

## ✅ 结果与验证

> 成功实现了多轮对话，Agent 能够：
> 1. 解析用户意图
> 2. 调用天气 API 获取实时数据
> 3. 根据天气推荐合适的景点
> 4. 用自然语言总结并返回给用户

## 🐛 问题与解决

### 问题 1：LLM 输出格式不稳定
- **现象**：有时 LLM 会输出多对 Thought-Action
- **原因**：模型生成不受约束
- **解决方案**：通过正则表达式截断，只保留第一对 Thought-Action
