# Superpowers MOC

> Superpowers 是一套完整的软件开发方法论，为 AI Agent 提供可组合的技能集，确保 Agent 在正确的时间使用正确的工作流。

---

## 什么是 Superpowers？

Superpowers 是一个**零依赖的插件系统**，为 Claude Code、Codex CLI、Cursor、GitHub Copilot 等 AI 编码工具提供结构化的软件开发方法论。

核心特点：
- **自动触发**：技能在正确的时刻自动激活
- **强制工作流**：不是建议，而是强制执行的流程
- **行为塑造**：通过精心设计的技能内容塑造 Agent 行为
- **可验证**：每个技能都经过压力测试和对抗性评估

---

## 八大核心工作流

1. [[Brainstorming]] — 苏格拉底式设计精炼
2. [[Writing Plans]] — 详细的实施计划
3. [[Subagent-Driven Development]] — 子代理驱动开发
4. [[Test-Driven Development]] — 测试驱动开发
5. [[Systematic Debugging]] — 系统化调试
6. [[Using Git Worktrees]] — 隔离工作空间
7. [[Requesting Code Review]] — 代码审查请求
8. [[Finishing a Development Branch]] — 完成开发分支

## 核心哲学

| 原则 | 含义 |
|------|------|
| **TDD 优先** | 先写测试，永远 |
| **系统化而非临时** | 流程优于猜测 |
| **降低复杂度** | 简单是首要目标 |
| **证据胜于声明** | 验证后再宣称成功 |
