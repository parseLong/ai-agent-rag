# 标签体系与使用指南

> 本文件定义了 AI Agent 第二大脑的标签体系，帮助你对笔记进行分类和检索。

---

## 🏷️ 标签层级

### 第一层：知识类型

| 标签 | 说明 | 示例 |
|------|------|------|
| `#概念` | 核心概念定义 | `#智能体`, `#大语言模型` |
| `#原理` | 底层原理 | `#Transformer原理`, `#注意力机制` |
| `#方法` | 方法论、技巧 | `#提示工程`, `#微调方法` |
| `#工具` | 工具、框架、平台 | `#LangChain`, `#Dify`, `#Coze` |
| `#实践` | 实战经验、项目 | `#项目实战`, `#代码实现` |
| `#案例` | 具体案例分析 | `#智能旅行助手`, `#赛博小镇` |

### 第二层：学习阶段

| 标签 | 说明 | 使用场景 |
|------|------|----------|
| `#学习笔记` | 学习过程中的记录 | 刚学完一章时的原始笔记 |
| `#待整理` | 需要进一步整理的笔记 | 内容较乱，需要提炼 |
| `#概念卡片` | 精炼的核心概念 | 经过验证的知识卡片 |
| `#永久笔记` | 经过验证的知识 | 经过多次更新、相对稳定的笔记 |
| `#项目` | 项目相关笔记 | 实践项目、代码实验 |
| `#日志` | 日常记录 | 每日笔记、复盘 |

### 第三层：主题领域

| 标签 | 说明 |
|------|------|
| `#智能体` | Agent 通用知识、架构、范式 |
| `#LLM` | 大语言模型相关 |
| `#记忆系统` | Memory、RAG、向量存储 |
| `#多智能体` | Multi-Agent 协作、通信 |
| `#评估` | Evaluation、Benchmark |
| `#训练` | Training、SFT、RLHF、GRPO |
| `#协议` | MCP、A2A、ANP 等通信协议 |
| `#上下文工程` | Context Engineering |
| `#低代码平台` | Dify、Coze、n8n 等 |
| `#框架` | LangGraph、AutoGen、AgentScope |

---

## 📝 标签使用规则

1. **每篇笔记至少一个标签**：确保笔记可检索
2. **标签层级清晰**：优先使用具体标签，如 `#ReAct` 而非仅 `#智能体`
3. **避免过度标签**：每篇笔记 3-5 个标签为宜
4. **定期整理标签**：检查并合并重复或相似标签

---

## 🔍 常用检索

### 按学习阶段检索

```
#学习笔记 #待整理    → 找到所有需要整理的笔记
#概念卡片            → 找到所有核心概念
#永久笔记            → 找到所有成熟的知识
```

### 按主题检索

```
#智能体 #概念        → 智能体相关的概念定义
#记忆系统 #实践      → 记忆系统的实践经验
#多智能体 #案例      → 多智能体的案例分析
```

---

## 🗂️ 标签维护

- **每月检查**：查看是否有重复标签
- **合并相似标签**：如 `#LLM` 和 `#大语言模型` 合并为 `#LLM`
- **删除未使用标签**：清理不再使用的标签
