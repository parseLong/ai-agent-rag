# 知识沉淀护城河

> ==工作流只是管道，知识才是流过管道的活水。== 模型会迭代，工具链会更新，工作流会重构。但团队在特定业务领域积累的领域模型、架构决策、最佳实践、已知陷阱、业务流程——这些知识是永恒的，不会因模型换代而失效。

> [!quote] 核心论断
> "将来的技术护城河不在模型，而在垂直领域知识的沉淀。"
> — Harness 圆桌讨论

> [!important] 核心洞察
> ==Skill、Agent、工具链会随模型迭代更新，但领域知识是永恒的。== AI Team 的每次交付都自动沉淀知识到团队共享仓库，所有成员共建共享，新工作流启动时自动站在前人肩上。

---

## 三个关键认知

### 1. 工作流是"可替换的"，知识是"可累积的"

今天用 16 阶段状态机编排，明天可能用图结构 DAG 编排。Agent 调度模式从串行到并行到分层级联，变化很快。甚至 SOTA 模型厂商会逐渐内化强化规划能力。但团队积累的知识——"广告预算扣减在高并发下会超扣，需用 Redis+Lua 保证原子性"——==不管工作流怎么变，都是有价值的==。

> [!note] 模型内化趋势
> Anthropic 的 Claude Code 本身就是极其纯粹的 harness 实现。其在 [Emotion concepts and their function in a large language model](https://www.anthropic.com/research/emotion-concepts-function) 论文中指出类似方向——可能未来的 Mythos 模型会通过探针系统 SAE 实现"情绪"稳定，从根本上实现 harness 希望解决的认知节省问题。

### 2. 没有知识沉淀的工作流是"一次性"的

反模式：团队搭了很复杂的 Agent 工作流，每次需求都跑全流程，但==每次从零开始==。上一次踩过的坑下次照踩不误，上一次的架构决策下次重新推导。这就是**没有知识闭环的工作流**——投入了工程成本搭建工具链，却没有让工具链变得越来越聪明。

### 3. 知识是团队的"复利资产"

知识分为三类：

| 类型 | 定义 | 获取难度 |
|------|------|----------|
| **散点型知识** | 孤立的事实 | 较易从模型获取 |
| **因果型知识** | A 导致 B 的推理链 | 中等，需实践积累 |
| **时空型知识** | 特定场景和时间窗口下才成立的经验 | ==最难，完全依赖团队实践== |

越是高阶的知识，越难以从模型中获得，越依赖团队实践积累。当知识库有成百上千条 proven 知识条目时，新成员、新项目都能"站在前人肩上"——==这就是知识的复利效应==。

## Big Model vs Big Harness 的务实立场

| 立场 | 说明 |
|------|------|
| 模型能力提升是大势所趋 | 知识工程架构应==对模型能力提升保持开放==——当模型更强时，同样知识被更好地利用 |
| 但模型能力提升==不能替代==领域知识 | 再强的模型也不知道你的业务系统里有哪些隐藏的坑 |
| 知识工程投入是==确定性回报== | 每沉淀一条 proven 知识，所有后续工作流都受益；模型能力提升的回报是概率性的 |
