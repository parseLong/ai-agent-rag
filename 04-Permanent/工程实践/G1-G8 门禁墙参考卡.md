# G1-G8 门禁墙参考卡

> 门禁墙是 Harness 稳定性支点的一半（另一半是 hook 拦截）。每个门禁是==确定性 Python 函数==，独立于上下文窗口运行，fail-closed（默认拒绝，只放行显式允许）。任一 gate FAIL → 流程退回 DEVELOPING——不是"建议"，是"阻断"。

---

## 八个门禁定义

| 门禁 | 代码 | 检查什么 | 检查方式 | FAIL 后果 |
|------|------|---------|---------|----------|
| **G1 产物存在性** | `gate_g1_artifacts` | 关键产物文件是否生成 | 文件路径检查（`phases/*.md` 是否存在） | 退回 DESIGNING |
| **G2 规则命中率** | `gate_g2_rules` | 必要规则是否被触发 | 预设规则命中率统计（按 intent×risk 裁剪） | 退回 DESIGNING |
| **G3 编译通过** | `gate_g3_compile` | 代码能否编译 | ==真跑== `mvn compile`，不信 AI 自报 | 退回 DEVELOPING |
| **G4 单测通过** | `gate_g4_unit_test` | 单元测试是否通过 | ==真跑== `mvn test`，编译失败 = 0 分 | 退回 DEVELOPING |
| **G5 代码覆盖率** | `gate_g5_coverage` | 测试覆盖是否达标 | 覆盖率阈值检查 | 退回 DEVELOPING |
| **G6 ATDD 通过** | `gate_g6_atdd` | 验收测试是否通过 | ==真跑== ATDD 测试 | 退回 DEVELOPING |
| **G7 接口测试通过** | `gate_g7_api_test` | 接口测试证据是否完整 | 部署预发后接口测试 + 证据检查 | 退回 DEPLOYING |
| **G8 生产上线确认** | `gate_g8_online` | 生产环境是否健康 | ==目前不强制==，人工兜底 | 人工确认 |

---

## 核心设计原则

### 1. 确定性代码，不依赖 LLM

> 流程强制执行必须从 LLM 推理中外置到确定性基础设施。不能依赖模型"记住"该执行哪个步骤——门禁必须是确定性代码，独立于上下文窗口。

这与 [[Agent 遗忘三重根因]] 中"指令遵循失败"根因直接对应：==如果 AI 不听话了怎么办？答案不是让它更聪明，而是不管听不听话，越界就被拦住==。

### 2. Fail-closed

默认拒绝，只放行显式允许的操作。`state-keeper` 单写者（hook 拦截其他写者）+ ajv schema 校验前置，从架构层面消除多 agent 写冲突。

### 3. 与 intent×risk 联动

不是所有需求都走全部门禁——按 intent×risk 动态裁剪：

| intent × risk | 必需门禁 |
|--------------|---------|
| `QUERY / NA` | 0 个 |
| `BUG_FIX · LOW` | G3 + G4 + 证据链 |
| `FEATURE · MEDIUM` | G1–G6 + 对抗辩论 |
| `FEATURE · HIGH` | G1–G7 拉满 + ADR |

## 与七维评分的关系

G1–G8 门禁与 [[七维确定性评分]] 的关系是==基础设施 vs 评分维度==：

| 七维评分维度 | 对应门禁 |
|------------|---------|
| 流程完整性（22%） | G1 + G2 |
| 代码正确性（22%） | G3 + G4 + G5 |
| 接口验收（18%） | G6 + G7 |
| 安全合规（8%） | 规则命中率（G2 的延伸） |

门禁是**运行时的硬阻断**，七维评分是**事后评估的量化维度**——两者检查的对象相同，但时序和力度不同。

---

## 与 [[Honesty Gap]] 的联动

G3 门禁（编译通过）引入了 **honesty gap** 检测——对比 AI 自报 `evidence.json` 的结果和真实编译结果：

```
verifier agent 写 verification.json → G3 检查
AI 自报"G3 通过" vs 真实编译结果 → honesty gap 暴露虚报
```

这意味着门禁不只检查"做了没有"，还检查"==说的是不是真的=="。

## 当前边界

**G8 生产上线不强制**——生产发布涉及灰度策略、流量切换、线上回归，目前"出错成本"远高于"效率收益"，由人兜底。

下一步目标：AI 自主完成预发验证 → 触发生产发布 → 观测线上指标 → 产出线上验收报告，把 G8 从"人工确认"进化为"AI 执行 + 人工兜底"。
