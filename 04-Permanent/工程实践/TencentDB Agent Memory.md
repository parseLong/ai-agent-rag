# TencentDB Agent Memory

> TencentDB Agent Memory 通过**上下文卸载 + Mermaid 无限画布**的组合方案，把长任务过程转化为可管理的结构化记忆：完整信息保留在外部，关键状态以高密度形式留存于上下文。在超长 Session 实验中，最高节省 61.38% Token，同时把任务通过率最多提升 51.52%。

## 核心架构

TencentDB Agent Memory 分两层解决两类"遗忘"问题：

| 记忆类型 | 解决的问题 | 核心方案 | 类比 |
|---------|----------|---------|------|
| **短期记忆压缩** | 任务进行中上下文变长、步骤变乱、目标丢失 | 上下文卸载 + Mermaid 无限画布 | 集中注意力完成当前任务 |
| **长期个性化记忆** | 下次再见时 Agent 不知道用户偏好、习惯和长期目标 | 用户偏好分层沉淀 + 按需注入 | 下次更懂你 |

## 短期记忆压缩：四级折叠架构

短期记忆压缩的核心公式：

**短期记忆压缩 = [[上下文卸载]] + [[Mermaid任务画布]]**

一次工具调用结束后，同一份信息被拆成三种形态，放到不同位置：

```
完整 tool result
  ↓
Level 0: refs/*.md          — 原始证据（完整原文，最重）
  ↓ 压缩
Level 1: offload-*.jsonl    — 工具调用级摘要（summary + result_ref）
  ↓ 聚合
Level 2: mmds/*.mmd         — 任务步骤级摘要（节点状态+summary+timestamp）
  ↓ 折叠
Level 3: Metadata           — 任务级索引（taskGoal, status, mmdFilePath）
```

| 层级 | 存储位置 | 信息粒度 | 回答的问题 | 上下文中的角色 |
|------|---------|---------|-----------|-------------|
| **Level 0** Raw | refs/*.md | 原始证据 | 完整工具结果是什么？ | 不常驻，按需读取 |
| **Level 1** JSONL | offload-*.jsonl | 工具调用级摘要 | 这一次具体工具调用做了什么？完整结果在哪里？ | 不常驻，按需 grep |
| **Level 2** MMD | mmds/*.mmd | 任务步骤级摘要 | 这个子任务推进到了哪里？得到什么阶段性结论？ | 当前任务常驻，历史任务可折叠 |
| **Level 3** Metadata | 上下文中 | 任务级索引 | 这里曾经有什么任务？当前状态？可从哪里恢复？ | 历史任务入口，始终保留 |

### 折叠与展开

信息根据上下文压力和任务状态，**逐级折叠**：

- **上下文压力不高时**：当前任务的 Active MMD 常驻上下文（Level 2）
- **上下文压力升高时**：历史任务的完整 MMD 折叠为 Metadata（Level 3）
- **需要细节时**：通过 [[层次化注意力机制]] 逐级下钻

**展开（找回）也是分级的**：

```
Metadata → mmdFilePath → 打开 MMD 画布
  ↓ 需要工具调用细节
MMD node_id → grep offload-*.jsonl → 找到 JSONL 记录
  ↓ 需要原始证据
JSONL result_ref → refs/*.md → 读取完整原文
```

> [!important] 关键设计
> MMD 节点本身已有摘要，所以 Agent 很多时候不需要继续下钻。只有当节点 summary 不够支持下一步判断时，才查 JSONL；只有当 JSONL summary 也不够时，才读 refs 原文。这避免了两种极端：所有信息塞在上下文里导致腐烂，或压缩得太狠导致频繁重新调用工具。

## 实验结果：超长 Session 压力测试

所有实验把多个任务串在同一 session 中连续执行（而非单题清空上下文的独立评测），更接近真实 Agent 使用场景：

| 评测集 | 任务类型 | 加插件后成功率相对变化 | Token 节省 |
|--------|---------|---------------------|-----------|
| **SWEbench** | 代码修复 | 完成率相对提升 9.93% (58.4%→64.2%) | 最高 33.09% |
| **Toolathlon** | 复杂长任务 | 通过率从 20%→35% (相对+75%) | 最高 26.18% |
| **WideSearch** | 网页搜索 | 通过率相对提升 51.52% | 最高 **61.38%** |
| **AA-LCR** | 长文总结与分析 | 准确率相对提升 7.95% (44%→47.5%) | 总 Token 节省 30.98% |

### 消融实验：MMD 的作用

| 方案 | SWEbench 成绩变化 | Token 节省 | 说明 |
|------|------------------|-----------|------|
| 无插件 | 基线 | 基线 | 完整工具结果留在上下文中 |
| **仅上下文卸载** | 提升约 5% | 节省约 15% | 原文移出上下文，但任务结构保留较弱 |
| **上下文卸载 + MMD** | 提升约 9.9% | 节省 31-33% | 原文可恢复，任务结构也被保留 |

> [!important] 消融实验结论
> [[上下文卸载]]解决"内容太长"，[[Mermaid任务画布]]解决"结构容易丢"。两者组合的效果远大于单独使用。

### 最适合的应用场景

短期记忆压缩尤其适合**办公提效、创作、研究和编程**这类长任务和多任务场景——共同特点是**资料多、步骤长、反复改**：

| 场景 | 对应评测集 | 压缩的价值 |
|------|-----------|-----------|
| 办公与行业研究 | WideSearch | 保留调研路径和阶段性结论，减少重复搜索 |
| 报告写作与创作 | AA-LCR | 记录版本调整、用户反馈、关键引用来源 |
| 编程与代码修复 | SWEbench | 保留 bug 定位路径、关键文件摘要、测试结果引用 |
| 复杂多步骤任务 | Toolathlon | 拆分子任务节点，知道哪些完成、哪些进行、哪些需恢复 |

## 长期个性化记忆

在 PersonaMem 评测集（6000+ 消息、589 道题）上：
- 回答准确率相对提升 **59%**（从不到 48% 提升到 76%）
- 部分 Token 节省场景接近 90%（但这类评测与真实使用方式存在差异）

长期记忆同样延续"渐进式披露"思想：不把所有用户历史一次性塞进上下文，而是将偏好、目标、习惯和经验分层沉淀，在合适时按需注入。
