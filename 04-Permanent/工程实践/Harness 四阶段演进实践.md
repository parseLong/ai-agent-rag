# Harness 四阶段演进实践

> 不是设计出来的，是被现实一路逼出来的。==真正推动架构演进的从来不是"想要更好"，而是"现在的做法已经崩了"。==每一阶段的切换都不是优化，是止损。

![[07-Attachments/harness-practice/四阶段演进图.jpg]]

核心转变只有一句话：==从"用更多的字约束 AI"，到"用更好的结构约束 AI"。==

---

## 第一阶段：拿来主义

用 `oh-my-claudecode`、`everything-claude-code` 等社区项目的 OpenSpec 规范直接上手——别人总结好的 CLAUDE.md 模板、多 agent 示例、最佳实践合集。

| 效果 | 局限 |
|------|------|
| 从"裸用 AI"跨进"有章法地用 AI" | 通用规范覆盖不了特定开发流程，边界情况全靠临场补丁 |

> 触发词：==每次我要写的额外 prompt 比规范本身还长时，就意味着该自己造了。==

---

## 第二阶段：重 prompt 约束

把所有流程规矩写进 CLAUDE.md——需求分析怎么做、方案设计几步走、编码完必须跑单测、部署前必须合 master……

**三天后崩了**。三个症状：

| 症状 | 机制 |
|------|------|
| **不听话** | 规则太多 → 模型"选择性遵守"：这次记得先写单测，下次又跳过 |
| **上下文爆炸** | 所有规则常驻窗口 → 留给代码的空间被挤压，"读完规则就没脑容量读代码" |
| **自我矛盾** | 规则间偶尔冲突（"快速修复走简化路径" vs "所有改动必须走评审"）→ 模型编造折中方案 |

![[07-Attachments/harness-practice/上下文爆炸示意图.jpg]]

> ==核心教训：prompt 约束是说服，不是强制==。模型"理解"了规则 ≠ "遵守"了规则——你无法用更多的字来对抗概率性的遗忘。

> [!note] 这正是 [[Agent 遗忘三重根因]] 中"指令遵循失败"根因的现场表现——规则都在上下文中，但模型仍然跳步。

---

## 第三阶段：减负 + 分层加载

问题的根因已经清晰：==把"有状态的流程"硬塞进了"无状态的对话窗口"==——本质上用错了工具。

做了两件事：

1. **给 harness 减负**：常驻 prompt 从"全流程指令手册"砍到只剩角色定义 + 触发规则，压到 ≤8K。深度内容移到 `context/` 层，只在对应阶段才 Read。
2. **整理三层加载链路**：常驻入口层 → 原子规则层 → 按需上下文层——把上下文当预算管理。

![[07-Attachments/harness-practice/减负分层效果.jpg]]

效果立竿见影：模型终于有"脑容量"理解代码了。

**新问题**：长程会话中，写了几百行代码后，规则虽还在但已被稀释到注意力衰减区——典型症状：写完代码后忘记该走什么流程。

> [!note] 这正是 [[上下文腐蚀与注意力衰减]] 的 U 型注意力分布效应——规则被几十屏代码输出挤到模型"看不见"的位置。

---

## 第四阶段：Agent 调度编排

最后一跳，认知上最大的转变：==不再约束模型"你该怎么做"，而是让不同的 agent 各司其职、互相制衡==。

![[07-Attachments/harness-practice/Agent调度编排.jpg]]

核心设计：dispatcher（流程驱动器）作为大脑，只负责"算下一步该谁上场"；其他 agent 各管一段。

一次高强度全天重构验证了这个架构：==状态外置、决策收敛给 dispatcher，即使单次会话崩了、上下文被压缩了，状态不丢、流程能续==。

### 24 agent 的过度拆分代价

- 每个 agent 的 system prompt 本身就是"小型 CLAUDE.md"，规则指令占满上下文
- agent 间转交多、调试链路长、维护心智负担大

后续合并 intent-classifier / debate-moderator / pre-mortem 等流程节点入主干 agent，精简冗余中间调度层——==保留核心约束（dispatcher 路由、职责隔离、状态外置、门禁阻断），降低协调成本==。

## 踩坑清单（每个已固化成规则）

| 坑 | 根因 | 教训 |
|---|------|------|
| headless 四连坑 | 无头环境里每个"理所当然"的东西都是地雷 | 必须显式接管浏览器、工作目录、进程生命周期 |
| 评分假"持平" | 空产物被兜底成 70 分中性值 | ==失败必须响亮==，绝不静默兜底 |
| 子 agent stall | 让它自由探索 270+ 文件直接卡死 | 给精确文件清单，别让它自己找 |
| 云端"找不到符号" | 分支没先合 master | 提预发前强制合主干 |
