# AgentScope

## 📖 定义

AgentScope 是阿里巴巴达摩院开发的多智能体开发平台，采用**消息驱动架构**和**组合式设计**。它提供了一套友好的编程接口，让开发者可以轻松定义智能体、构建通信网络，并管理整个应用的生命周期。其内置的消息传递机制和对分布式部署的支持，使其非常适合构建和运维复杂、大规模的多智能体系统。

目前 AgentScope 已覆盖 Python、TypeScript、Java 三种语言，均升级至 2.0 版本。**Java 2.0 版本**进一步面向 JVM 生态与企业级生产场景，聚焦真实场景下的稳定运行、安全控制、分布式部署与接入需求。

## 🧠 我的理解

AgentScope 的设计哲学是"工程化优先"。如果说 AutoGen 更像是一个灵活的"对话工作室"，那么 AgentScope 就是一个完整的"智能体操作系统"。它不仅关注如何构建智能体，更关注如何在大规模、高并发的生产环境中稳定运行。

Java 2.0 延续了 1.0 的"透明开发"理念，把"让智能体在企业环境中可靠运行"做成框架内生能力。对企业用户而言，让智能体跑通一次通常不是问题，难的是让它**长期稳定地运行、支持分布式部署与多租户隔离**。

## 🎯 为什么重要

AgentScope 代表了多智能体系统从"实验原型"到"生产应用"的跨越。它解决了工程化中的核心问题：并发、容错、分布式部署、可观测性。对于需要构建企业级多智能体系统的开发者来说，AgentScope 是目前最值得关注的框架之一。

## 🔑 分层架构

| 层级           | 组件                            | 说明         |
| ------------ | ----------------------------- | ---------- |
| **基础组件层**    | Message、Memory、Model API、Tool | 核心构建块      |
| **智能体基础设施层** | ReAct、Hooks、并行工具调用            | 高级抽象       |
| **多智能体协作层**  | MsgHub、Pipeline               | 消息路由和工作流编排 |
| **开发部署层**    | Runtime、Studio                | 运行时和可视化工具  |

## 🔑 关键特性（1.0）

- **消息驱动架构**：所有交互抽象为消息的发送和接收，异步解耦
- **位置透明**：智能体无需关心对方在本地还是远程
- **可观测性**：每条消息可记录、追踪、分析
- **原生分布式**：支持跨节点通信，对开发者透明
- **结构化输出**：通过数据模型约束智能体行为

## 🏢 Java 2.0 企业级升级

![[Assets/agentscope-java2-architecture.png]]

AgentScope Java 2.0 是面向企业级需求的一次系统性升级，核心升级点涵盖以下七大领域：

### 1. HarnessAgent：长期稳定运行的工程封装

**HarnessAgent** 是 `ReActAgent` 之上的工程化封装——核心 ReAct 推理循环原样保留，但围绕长期稳定运行所需的工作区、长期记忆、上下文压缩、子智能体编排、沙箱隔离、计划模式等工程能力，全部打包进单一 builder。

```java
HarnessAgent agent = HarnessAgent.builder()
    .name("demo-agent")
    .model("dashscope:qwen-max")
    .workspace(Paths.get(".agentscope/workspace"))
    .filesystem(new DockerFilesystemSpec()
        .isolationScope(IsolationScope.USER))
    .build();
agent.call(msg, RuntimeContext.builder()
    .sessionId("demo").userId("alice").build()).block();
```

**设计目标**：ReActAgent 提供 Agent loop 抽象与底层原子能力，Harness 提供保障效果、稳定、分布式部署、长期运行的一站式解决方案——只叠加，不替换。开发者从 ReActAgent 起步，需要长期运行时无缝迁移到 HarnessAgent，业务代码无须改动。

### 2. Workspace 与抽象文件系统：执行环境与逻辑解耦

![[Assets/agentscope-java2-workspace.png]]

AgentScope Java 2.0 把智能体的资源管理拆成两层正交的抽象：

- **工作区（逻辑视图）**：把智能体长期运行所需的全部资源——人格设定（AGENTS.md）、长期记忆（MEMORY.md）、领域知识、可复用技能、子智能体声明、工具与 MCP 白名单、会话快照与对话日志——统一组织成标准化目录结构。每轮推理时框架按需拼进 system prompt，开发者把工作区版本化进 Git，改文件即升级智能体。

- **抽象文件系统（物理载体）**：同一份工作区目录布局，背后可挂三类存储后端，部署时切换即可：

| 存储后端 | 适用场景 | 特点 |
|----------|----------|------|
| **本地文件系统** | 开发与单机部署 | 零配置开箱即用，直接读写宿主磁盘 |
| **沙箱文件系统** | 企业多租户 / 不可信输入 | 容器隔离执行，状态可按会话/用户粒度持久化到对象存储或 Redis，节点切换后可从快照重建 |
| **远端文件系统** | 纯 Web 服务 / 无需本地 shell | 直接落到远端 KV 或对象存储，多副本共享同一逻辑工作区 |

**关键价值**：开发 → 测试 → 生产的三段路径不需要改代码，同一份智能体实现，按环境切换底层存储即可自由迁移。

### 3. Context：长期任务的上下文管理机制

![[Assets/agentscope-java2-context.png]]

上下文管理在 2.0 中升级为支撑长期任务执行的**系统策略**，而非简单的"压缩历史"：

- **结构化压缩**：压缩结果不是简单摘要，而是结构化保留任务目标、当前状态、关键发现、下一步计划
- **超大工具结果自动卸载**：git diff、mvn test 输出等几十 K 字符的内容自动卸载到工作区，上下文只保留首尾摘要 + `read_file` 路径占位符
- **文件缓存机制**：内置文件读写工具加入缓存，减少重复 IO，编辑前必须先读取以提升可靠性
- **兜底压缩重试**：撞到 `context_length_exceeded` 时框架自动触发兜底压缩，避免任务直接断在边界处

### 4. 模型接入：开放生态 + 容错能力

![[Assets/agentscope-java2-model.png]]

2.0 继续保持开放模型接入（Qwen、Anthropic、DeepSeek、Gemini、OpenAI、Grok、Moonshot、Ollama 等），重点升级在于**稳定性**：

- **统一 Credential + ChatModel 抽象**：每个厂商同一套 builder 后面的独立实现
- **FallbackModel 备用模型机制**：主模型不可用、限流或过载时框架自动透明切换到备用模型链，保持任务执行连续性

### 5. 消息与事件：可交互执行流

![[Assets/agentscope-java2-event.png]]

- **ContentBlock 统一消息类型**：一条消息可同时包含文本、图片、文件、工具调用/结果、模型思考、用户确认状态等。Java 侧借助 sealed class 与 record 表达强类型，非法 role × content 组合在构造期就被拦下。DataBlock 同时兼容 base64 与 URL 两类数据源。
- **事件系统**：`call()` 不再只返回最终文本，`streamEvents()` 流式产生类型化事件（模型调用开始、文本增量、工具调用、用户确认、外部执行等），基于 Project Reactor 的 Flux 输出，前端 UI 直接订阅即可实时跟随。
- **人工确认与介入**成为框架内生能力，而非额外实现。

### 6. 权限系统：自主执行的边界

![[Assets/agentscope-java2-permission.png]]

- **三态决策**：工具调用不再是简单"允许/禁止"，而是"允许 / 用户审批（HITL）/ 拒绝"
- **综合判断维度**：基于静态规则、工具类型和输入内容综合判断
  - 文件读写工具检查危险目录和敏感文件
  - 命令执行工具分析高风险命令、动态 shell 结构和危险删除操作
- 未知或高风险行为自动进入用户审批流程，决策权交回给人

### 7. Middleware：框架扩展机制

![[Assets/agentscope-java2-middleware.png]]

把原本松散的 Hook 列表收敛成**五个清晰阶段**：

| 阶段 | 关注点 |
|------|--------|
| **onAgent** | Agent 生命周期 |
| **onReasoning** | 推理流程中的业务策略 |
| **onActing** | 工具执行前的安全检查 |
| **onModelCall** | 模型调用前后：日志追踪、限流 |
| **onSystemPrompt** | 动态上下文注入 |

每个关注点各居其层，组合干净利落，保持核心框架稳定的同时为不同应用场景留下灵活扩展空间。

## 🏢 分布式部署与多租户

Java 2.0 把分布式部署做成一等公民：

1. **分布式会话与沙箱管理**：单机开发阶段会话状态默认落到 workspace 目录，零配置；生产部署时替换为分布式存储，任意副本都能拉到完整快照接续工作。沙箱模式下容器内积累的环境会在每次调用结束时打包成快照落到对象存储或 Redis，节点切换后可从快照重建。框架在装配阶段校验配置一致性，防止上线后发现状态丢失。

2. **多租户隔离贯穿执行链路**：RuntimeContext 的 `userId` / `sessionId` 直接贯穿工作区路径、KV 命名空间、沙箱环境，参与每一次资源寻址。开发者按业务语义选择隔离粒度——对话级隔离、用户级共享、公共工具型共享或全局共享——框架通过系统强制约束而非依赖业务代码自觉。

3. **统一文件系统层**：所有文件操作收敛到 `AbstractFilesystem` 抽象，每次调用自动带上会话与用户身份信息，框架据此隔离到对应租户命名空间。三类后端共用同一套上层语义，部署切换不需要改代码。

## 💡 实例/案例

在 Hello-Agents 教程中，使用 AgentScope 构建了一个"三国狼人杀"游戏：
- 多个智能体扮演不同角色（刘备、关羽、诸葛亮等）
- 消息中心（MsgHub）管理智能体间的私密和公开通信
- 结构化输出约束游戏规则（如女巫无法同时对同一目标使用解药和毒药）
- 展示了并发处理、容错机制、角色建模等高级特性

## 📝 个人笔记

AgentScope 的工程化能力是其最大亮点，但也带来了更高的学习曲线。它的消息驱动架构虽然强大，但需要开发者理解异步编程和分布式通信。对于需要构建大规模、高可靠性系统的场景，AgentScope 是不二之选；但对于快速原型开发，AutoGen 或 CAMEL 可能更轻量。

Java 2.0 的升级方向非常清晰——所有改动都指向"让智能体在企业环境中可靠运行"。HarnessAgent 的"只叠加、不替换"设计哲学值得注意，它让开发者可以按需从 ReActAgent 渐进升级，而不需要重写业务代码。Workspace 与抽象文件系统的两层切分也是企业落地时非常实用的设计——同一份智能体实现跨环境跑不再只是口号，而是默认行为。
