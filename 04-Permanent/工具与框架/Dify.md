# Dify

## 📖 定义

Dify 是一个开源的大语言模型（LLM）应用开发与运营平台，融合了后端即服务（BaaS）和 LLMOps 理念，为从原型设计到生产部署提供全流程支持。它采用分层模块化架构，支持 Agent 工作流、RAG Pipeline、数据标注与微调等多种能力，是目前最成功的智能体平台之一。

## 🧠 我的理解

Dify 是一个面向开发者和企业的"全栈式"智能体开发平台。与 Coze 的零代码定位不同，Dify 在低代码开发的便利性和专业开发的灵活性之间取得了良好平衡。它的核心优势在于：模型中立（支持数百种开源或专有 LLM）、强大的插件生态（8000+ 插件）、企业级安全（AES-256 加密、RBAC 权限控制），以及灵活的部署方式（本地部署 + 云端 SaaS）。

## 🎯 为什么重要

Dify 代表了智能体开发从"玩具"走向"生产"的关键一步。它不仅是低代码平台，更是一个完整的 AI 应用操作系统，让开发者能够构建可扩展、可维护、安全合规的企业级智能体应用。

## 🔑 核心特性

| 特性 | 说明 |
|------|------|
| **全栈开发** | 整合 RAG 管道、AI 工作流、模型管理等功能 |
| **模型中立** | 支持 GPT、DeepSeek、Llama 等数百种模型 |
| **插件生态** | Marketplace 拥有 8000+ 插件，涵盖各种功能 |
| **企业安全** | AES-256 加密、RBAC 权限控制、审计日志 |
| **灵活部署** | 支持 Docker 本地部署和云端 SaaS |
| **MCP 支持** | 原生支持 Model Context Protocol 协议 |

## 🔑 适用场景

- **企业级应用**：需要复杂业务逻辑和数据安全的场景
- **多智能体系统**：需要问题分类器和智能路由的复杂架构
- **数据处理**：数据查询、可视化分析、SQL 生成
- **多模态应用**：图片、视频生成等富媒体场景

## 💡 实例/案例

在 Hello-Agents 教程中，使用 Dify 构建了一个"超级智能体个人助手"，涵盖：
- 日常生活问答
- 文案润色优化
- 多模态内容生成（图片、视频）
- 数据查询与可视化分析
- MCP 工具集成（高德地图、饮食推荐、新闻资讯）

### 案例详解：超级智能体个人助手

**多智能体架构**：使用问题分类器进行智能路由，将用户请求分发到对应的处理模块。

**各模块提示词示例**：

1. **日常助手模块**（基础对话兜底）：
```
# Role: 日常问题咨询专家
## Profile
- language: 中文
- description: 专门回答用户日常生活中的一般性问题
- expertise: 日常生活、健康养生、家庭管理、人际关系

## Rules
- 实用性优先: 确保提供的建议具有可操作性
- 准确性保证: 基于可靠信息和常识给出回答
- 中立客观: 避免个人偏见和主观臆断
```

2. **文案优化模块**：
```
# 一、角色人设（Role）
你是一位专业的文案优化专家，擅长提升文案的吸引力、转化率和可读性。

# 二、任务目标（Task）
- 分析并优化文案的结构、语言和风格
- 提升文案的吸引力、可读性和转化潜力
- 根据常见优化原则（简洁性、情感共鸣、行动号召等）进行调整
```

3. **数据分析模块**：
```
# 一、角色人设（Role）
你是一位专业的数据分析师，具备数据整理、清洗和可视化能力。

# 三、任务目标（Task）
1. 对原始数据进行归纳和整理
2. 按照正确的逻辑对数据进行分类和排序
3. 数据展示突出关键信息和数据洞察
4. 提供易于理解的数据展示
```

**MCP 集成步骤**：
1. 在魔搭社区 MCP 市场选择 hosted 类型（如高德 MCP）
2. 选择 SSE 模式并生成 MCP 配置 JSON
3. 在 Dify 中选择支持 MCP 调用的智能体策略（ReAct 模式）
4. 配置 MCP 服务（删除 `mcp-server` 前缀，选择 SSE 模式）

## 📝 个人笔记

Dify 的插件生态是其最大护城河。与 Coze 相比，Dify 支持 MCP 协议，这意味着它可以无缝接入任何遵循 MCP 标准的工具和服务。对于需要深度定制的企业来说，Dify 的开源属性也是一个重要优势。
