# n8n

## 📖 定义

n8n 是一个开源的工作流自动化平台，通过可视化节点连接实现复杂的业务流程自动化。与 Coze 和 Dify 不同，n8n 本质上是一个通用自动化工具，而非纯粹的 LLM 平台。近年来，它积极集成 AI 能力，使得在自动化流程中嵌入大语言模型成为可能。

## 🧠 我的理解

n8n 的核心是"连接"。它拥有数百个预置节点，可以将各类 SaaS 服务、数据库、API 连接成复杂的自动化流程。AI 只是这个流程中的一个（或多个）强大的"处理节点"。这种设计让 n8n 特别适合将 AI 能力深度整合进现有业务流程。

## 🎯 为什么重要

n8n 代表了"AI + 自动化"的深度融合。当其他平台还在思考"如何构建 AI 应用"时，n8n 已经在解决"如何让 AI 自动执行现有业务流程"。对于需要高度定制化自动化、注重数据隐私的企业来说，n8n 的私有化部署能力尤为重要。

## 🔑 核心概念

| 概念 | 说明 |
|------|------|
| **节点 (Node)** | 执行具体操作的最小单元，分为触发节点和常规节点 |
| **工作流 (Workflow)** | 由多个节点连接而成的自动化流程图 |
| **AI Agent 节点** | 整合模型、记忆和工具的高级节点 |
| **Simple Vector Store** | 内存中的向量存储，用于知识库 |
| **Simple Memory** | 内存中的对话记忆 |

## 🔑 核心优势

- **强大的连接能力**：数百个预置节点，连接各类服务
- **AI Agent 节点**：将模型、记忆和工具管理高度整合
- **私有化部署**：支持自托管，保障数据安全
- **可视化调试**：流程图直观展示数据流向
- **代码灵活性**：Code 节点支持自定义逻辑

## ⚠️ 局限性

- **存储非持久**：Simple Memory 和 Vector Store 基于内存，重启后数据丢失
- **调试繁琐**：复杂工作流的问题定位较困难
- **版本控制**：工作流的版本管理不如代码成熟（导出 JSON 后 diff 不清晰，多人编辑易冲突）
- **学习曲线**：通用自动化能力强大但上手较难

## 💡 实例/案例

在 Hello-Agents 教程中，使用 n8n 构建了一个"智能邮件助手"：
- 接收 Gmail 邮件触发工作流
- AI Agent 分析邮件内容，判断用户意图
- 调用 SerpAPI 搜索公开信息
- 查询私有知识库（工作时间、回复策略）
- 自动生成并发送回复邮件

### 案例详解：智能邮件助手

**整体架构**：`接收 -> AI Agent (思考 -> 决策 -> 工具调用) -> 回复`

**步骤1：构建私有知识库**

在 n8n 中使用 `Simple Vector Store` 节点构建知识库：

1. **定义知识源**（Code 节点）：
```javascript
return [
  {
    "doc_id": "work-schedule-001",
    "content": "我的工作时间是周一至周五，上午9点到下午5点。"
  },
  {
    "doc_id": "off-hours-policy-001",
    "content": "在非工作时间（包括周末和公共假期），我无法立即回复邮件。"
  }
];
```

2. **文本向量化**：使用 `Embeddings Google Gemini` 节点将文本转换为向量

3. **存入向量存储**：使用 `Simple Vector Store` 节点，Operation Mode 设为 `Insert Documents`

**步骤2：构建 Agent 主工作流**

1. **Gmail 触发器**：Event 设为 `Message Received`，配置邮箱账号

2. **AI Agent 节点配置**：
   - **Chat Model**：Google Gemini Chat Model
   - **Memory**：Simple Memory 节点（对话历史）
   - **Tools**：SerpAPI（搜索） + Simple Vector Store（私有知识库）

3. **Prompt 设计**（System Message）：
```
你是一个全天候待命、专业高效的AI邮件助手。

# 可用工具
- Simple Vector Store2: 查询我的个人信息（工作时间、回复策略）
- SerpAPI: 优先使用此工具在互联网上搜索答案

# 执行步骤
1. 分析邮件内容，提炼核心问题
2. 并行搜集信息：SerpAPI 搜索 + Vector Store 查询
3. 草拟核心回复
4. 添加状态前缀（工作时间 vs 非工作时间）
5. 以 JSON 格式输出最终结果
```

4. **发送回复**：使用 Gmail 节点的 `Send` 操作，通过 n8n 表达式关联收件人、主题和正文

## 📝 个人笔记

n8n 最适合的场景是"让 AI 接管现有业务流程"。比如：自动回复客户邮件、定时生成数据报告、跨系统数据同步等。它的价值不在于"构建 AI 应用"，而在于"用 AI 自动化业务"。
