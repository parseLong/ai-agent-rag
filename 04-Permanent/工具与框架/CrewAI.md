# CrewAI

## 📖 定义

CrewAI 是一个专注于构建多智能体团队的框架。它通过定义**角色（Roles）**、**任务（Tasks）**和**团队（Crew）**，让多个智能体协同工作以完成复杂目标。CrewAI 的设计哲学是"让专业的人做专业的事"——每个智能体被赋予特定角色和工具，通过结构化的协作流程共同完成任务。

## 🧠 我的理解

CrewAI 的核心理念是"团队化协作"。它将复杂的任务分解为多个子任务，分配给不同角色的智能体。与 AutoGen 的自由对话不同，CrewAI 更强调结构化的任务分配和执行流程，类似于企业中的项目管理方式。

## 🎯 为什么重要

CrewAI 提供了一种结构化的多智能体协作模式，特别适合需要将复杂任务分解并分配给不同"专家"的场景。它的设计让多智能体协作变得可预测、可管理，降低了构建复杂多智能体系统的门槛。

## 🔑 核心概念

| 概念 | 说明 |
|------|------|
| **Agent（智能体）** | 具有特定角色、目标和背景的智能体 |
| **Task（任务）** | 需要智能体执行的具体工作，可分配给特定智能体 |
| **Crew（团队）** | 智能体和任务的集合，定义协作流程 |
| **Process（流程）** | 任务的执行顺序，支持顺序和层次两种模式 |

## 🔑 关键特性

- **角色定义**：为每个智能体定义明确的角色、目标和背景
- **任务分配**：将任务分配给特定智能体，确保专业对口
- **协作流程**：支持顺序执行和层次化执行两种模式
- **工具集成**：每个智能体可配置专属工具集
- **结果输出**：支持多种输出格式（JSON、文本等）

## 💡 使用示例

```python
from crewai import Agent, Task, Crew

# 定义研究员智能体
researcher = Agent(
    role='研究员',
    goal='收集和分析信息',
    backstory='你是一位经验丰富的研究员',
    tools=[search_tool]
)

# 定义写作智能体
writer = Agent(
    role='作家',
    goal='撰写高质量的报告',
    backstory='你是一位专业的技术作家'
)

# 定义任务
research_task = Task(
    description='研究AI最新发展趋势',
    agent=researcher
)

write_task = Task(
    description='基于研究结果撰写报告',
    agent=writer
)

# 组建团队
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, write_task],
    process='sequential'
)

# 执行任务
result = crew.kickoff()
```

## 📝 个人笔记

CrewAI 的优势在于其结构化的协作模式，让多智能体系统的行为更可预测。相比 AutoGen 的"自由对话"，CrewAI 更像"项目管理"。适合需要严格任务分工和结果输出的场景。但也要注意，过于结构化的设计可能限制智能体的创造性和灵活性。
