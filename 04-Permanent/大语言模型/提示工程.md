# 提示工程

## 📖 定义

提示工程（Prompt Engineering）是研究如何设计精准的**提示（Prompt）**，从而引导大语言模型产生我们期望输出的技术。提示是我们与 LLM “大脑”沟通的语言，提示工程就是优化这种沟通方式的艺术。

## 🧠 我的理解

如果把 LLM 比作一个能力极强但“有点迷糊”的天才，那么提示工程就是学习如何与它有效沟通的技巧。同样的模型，不同的提示可能得到截然不同的结果。好的提示工程能让模型的性能提升数倍。

核心思想：**清晰、具体、结构化的输入 → 更好的输出**。

## 🎯 为什么重要

在智能体开发中，提示工程是核心技能：

- **零成本提升性能**：不需要训练模型，仅通过优化提示就能显著提升效果
- **决定智能体行为**：Agent 的“人格”和能力很大程度上由系统提示定义
- **影响多智能体协作**：清晰的提示能让智能体之间高效分工协作
- **降低幻觉风险**：结构化的提示可以引导模型减少编造

## 🎛️ 模型采样参数

在使用大模型时，`Temperature`、`Top-k`、`Top-p` 等参数通过调整模型对概率分布的采样策略，让输出匹配具体场景需求。

### Temperature（温度）

温度是控制模型输出”随机性”与”确定性”的关键参数。引入温度系数 $T > 0$，将 Softmax 改写为：

$$p_i^{(T)} = \frac{e^{z_i / T}}{\sum_{j=1}^k e^{z_j / T}}$$

| 温度范围               | 特点               | 适用场景                |
| ------------------ | ---------------- | ------------------- |
| 低温度（0 ≤ T < 0.3）   | 分布更”陡峭”，输出更精准、确定 | 事实性任务、代码生成、法律条文解读   |
| 中温度（0.3 ≤ T < 0.7） | 输出平衡、自然          | 日常对话、邮件撰写、产品文案      |
| 高温度（0.7 ≤ T < 2）   | 分布更”平坦”，输出更创新、发散 | 诗歌创作、科幻故事、广告 slogan |

### Top-k 采样

将所有 token 按概率从高到低排序，取排名前 k 个的 token 组成”候选集”，随后对筛选出的 k 个 token 的概率进行归一化：

$$\hat{p}_i = \frac{p_i}{\sum_{j \in \text{候选集}} p_j}$$

当 k=1 时输出完全确定，退化为”贪心采样”。

### Top-p 采样（核采样）

将所有 token 按概率从高到低排序，从排序后的第一个 token 开始，逐步累加概率，直到累积和首次达到或超过阈值 p：

$$\sum_{i \in S} p_{(i)} \geq p$$

此时累加过程中包含的所有 token 组成”核集合”，最后对核集合进行归一化。

### 参数协同工作

当同时设置 Top-p、Top-k 和温度系数时，这些参数会按照分层过滤的方式协同工作，其优先级顺序为：**温度调整 → Top-k → Top-p**。

> 注意：如果将温度设置为 0，则 Top-k 和 Top-p 将变得无关紧要，因为最有可能的 Token 将成为下一个预测的 Token。

## 🔑 核心技巧

### 1. 零样本、单样本与少样本提示

- **Zero-shot**：不给示例，直接下达指令。模型在海量数据上预训练后获得的强大泛化能力使其能够直接完成任务。

- **One-shot**：提供一个完整的示例，向模型展示任务的格式和期望的输出风格。

- **Few-shot**：提供多个示例，让模型更准确地理解任务的细节、边界和细微差别。

### 2. 思维链（Chain-of-Thought, CoT）

对于需要逻辑推理、计算或多步骤思考的复杂问题，在提示中加入引导语如”请一步一步地思考”或”Let's think step by step”，引导模型”一步一步地思考”。

### 3. 角色扮演（Role-playing）

通过赋予模型一个特定的角色，引导它的回答风格、语气和知识范围，使其输出更符合特定场景的需求。

### 4. 上下文示例（In-context Example）

通过在提示中提供清晰的输入输出示例，来”教会”模型如何处理请求，尤其是在处理复杂格式或特定风格的任务时非常有效。

## 🧩 指令调优的影响

**指令调优（Instruction Tuning）** 是一种微调技术，它使用大量”指令-回答”格式的数据对预训练模型进行进一步的训练。经过指令调优后，模型能更好地理解并遵循用户的指令。

- **对”文本补全”模型的提示**：需要用少样本提示”教会”模型做什么
- **对”指令调优”模型的提示**：可以直接下达指令

指令调优的出现，极大地简化了我们与模型交互的方式，使得直接、清晰的自然语言指令成为可能。

## 💡 实例/案例

**情感分类任务的提示设计**：

```
请判断以下文本的情感色彩（正面、负面或中性）。

示例：
文本：这家餐厅的服务太慢了。
情感：负面

文本：Datawhale的AI Agent课程非常棒！
情感：正面

文本：这部电影的情节很平淡。
情感：中性

现在请判断：
文本：这次旅行让我感到非常开心和满足。
情感：
```

## 📝 个人笔记

提示工程不是“ trick”，而是对 LLM 工作方式的深刻理解。在智能体开发中，系统提示的设计往往决定了 Agent 的上限。好的提示应该：明确角色、提供上下文、给出示例、设定边界、引导思考。
