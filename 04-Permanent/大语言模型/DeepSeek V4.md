# DeepSeek V4

> 2026 年 4 月 24 日开源的 DeepSeek 第四代模型，从 attention 到 kernel 的系统级重构——让 1M 上下文真正落地，开源最强。

## 模型规格

| 模型 | 总参数 | 稀疏激活 | 上下文 | 定位 |
|------|--------|----------|--------|------|
| DeepSeek-V4-Pro | 1.6T | 49B | 1M | 旗舰版，算力充足时的首选 |
| DeepSeek-V4-Flash | 284B | 13B | 1M | 高效版，算力有限时的性价比之选 |

> [!note] 两个常见误解
> - **Flash 不是 Pro 的蒸馏小模型**——两档都是独立预训练的 [[MoE架构]]，只是规模和稀疏度不同
> - **1M 上下文不是开关项**——两档都默认 1M，服务端不再区分"长/短"模型

## 为什么需要 1M 上下文？

1M 上下文不是"能写更长的 prompt"，而是让 Agent、整库代码、长文档等任务**真正落地执行**：

| 场景 | Token 消耗 | 传统方案的问题 |
|------|-----------|--------------|
| Agent 30轮 Coding | 数十万起步 | 128K 窗口撑不住几轮，V3.2 丢 thinking history 是因为**留不下** |
| 整仓库代码理解 | 80–120万 | RAG 漏一个调用点就是一个 bug，超长上下文让模型**自己看见所有调用关系** |
| 长文档推理 | 50–80万 | 切块摘要前后逻辑缺失，直接在原始材料上**跨段落推理** |

> [!important] 1M 是需求驱动，不是军备竞赛
> 当我们自己部署 OpenClaw 或 Hermes 成为助手时，发现还没问几个问题就已消耗几十万甚至上百万 token。128K 上下文真的撑不住几轮问题——1M 是 Agent 时代的刚需。

## 三项架构核心创新——系统级互锁

传统 Transformer 突破到 1M 上下文必须同时解决三个问题。这三项创新**不是孤立存在的**，它们之间有着严密的互锁关系：

| 问题 | 创新回应 | 互锁关系 |
|------|---------|---------|
| ① 残差通道僵化、深层贡献稀释 | [[mHC（多流约束残差连接）]] — 多流+双随机矩阵约束 | mHC 让更深的[[MoE架构]]网络训练稳定 → 稳定的深层网络才能承受 [[注意力机制]] CSA/HCA 的压缩损失 |
| ② O(L²) 计算量 + KV Cache 膨胀 | [[注意力机制]] CSA/HCA — "海选→精选→精确计算" | CSA/HCA 降低计算量和 KV Cache → 让 1M 上下文算得过、存得下 → 但需要 [[Muon优化器]] 保证深层网络训练稳定 |
| ③ 万亿参数深层网络训练不稳定 | [[Muon优化器]] — 梯度正交化 + RMSNorm 前置 | Muon 与 mHC 双保险 → 稳定的训练让 1T+参数的 [[MoE架构]] + CSA/HCA + mHC **全部能训练出来** |

> [!important] 系统级闭环的逻辑链条
> **mHC**（结构稳定性）→ 让深层 [[MoE架构]] 可训练 → **CSA/HCA**（计算效率）→ 让 1M 上下文算得过 → **Muon**（优化器稳定性）→ 让整个系统训练收敛 → 三者缺一则整个闭环断裂。
>
> 这就是 V4 "系统级闭环"的真正含义——不是"每一块都精进"，而是"每一块都依赖其他块才能起效"。

## V4 vs V3.2 的演进

| 维度 | V3.2 | V4 | 变化本质 |
|------|------|-----|---------|
| 残差机制 | 标准残差 | [[mHC（多流约束残差连接）]] | 从"单通道均等"到"多通道有权重+约束" |
| 注意力机制 | DSA | [[注意力机制]] CSA/HCA 混合 | 从"稀疏注意力"到"压缩+稀疏+高度压缩三层" |
| 优化器 | Adam | [[Muon优化器]] | 从"梯度方向可能互相干扰"到"正交化独立更新" |
| 上下文窗口 | 128K | 1M | 从"Agent 几轮就满"到"30轮 Coding 起步" |
| KV Cache | 标准 PagedAttention | [[KV 缓存]] 异构架构 | 从"假设所有层KV一致"到"每种注意力有自己Cache规则" |

## Infra 层优化

架构创新需要精细化 Infra 方案来"让硬件吃得饱"：

| 优化 | 原子笔记 | 说明 |
|------|---------|------|
| 算子开发 | [[TileLang]] | 数据流逻辑与调度策略解耦，编译性能超手工CUTLASS/cuBLAS |
| 训练可复现 | [[批无关性与计算确定性]] | bit级一致输出 + DeepGEMM + 加法括号结构钉死 |
| 部署效率 | [[FP4量化感知训练]] | MoE→FP4、CSA索引→FP4、打分→BF16，分模块精准量化 |
| 训练/推理框架 | [[DeepSeek V4 训练与推理框架优化]] | Muon兼容分布式、mHC流水线调优(额外6.7%)、跨机器压缩、单tensor级重算 |
| KV Cache | [[KV 缓存]] | CSA/HCA+SWA异构KV + 分类持久化 + SWA三档取舍 |
| MoE通信 | [[MoE架构]] | 细粒度计算通信重叠 |
