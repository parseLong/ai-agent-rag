# GPT

## 📖 定义

GPT（Generative Pre-trained Transformer，生成式预训练 Transformer）是由 OpenAI 提出的一系列大语言模型。其核心思想是：**语言的核心任务就是预测下一个最有可能出现的词**。GPT 完全抛弃了 Transformer 的编码器部分，只保留解码器，采用自回归方式生成文本。

## 🧠 我的理解

GPT 的哲学非常简单：给我一个开头，我来预测下一个词，然后把预测的词加回去继续预测，直到生成完整的文本。这种“文字接龙”的方式，让它能够胜任写作、对话、翻译、代码生成等各种任务。

## 🎯 为什么重要

- **开启了生成式 AI 时代**：GPT-3 的发布让大语言模型走进大众视野
- **通用人工智能的雏形**：一个模型可以处理多种任务
- **智能体的基石**：现代 AI Agent 几乎都基于 GPT 或类似架构
- **引发了 AI 革命**：ChatGPT 的爆火让 AI 进入千行百业

## 🔑 核心技术特点

### 1. Decoder-Only 架构

- 完全抛弃编码器，只保留解码器
- 使用**掩码自注意力（Masked Self-Attention）**：在预测第 $t$ 个词时，应用掩码阻止模型关注位于当前位置之后的信息
- 自回归生成：一次生成一个词，将刚刚生成的词添加到输入文本的末尾，再次预测下一个词

**Decoder-Only 架构的优势**：
- **训练目标统一**：模型的唯一任务就是"预测下一个词"，非常适合在海量的无标注文本数据上进行预训练
- **结构简单，易于扩展**：更少的组件意味着更容易进行规模化扩展
- **天然适合生成任务**：其自回归的工作模式与所有生成式任务（对话、写作、代码生成等）完美契合

### 2. 预训练 + 微调范式

- **预训练**：在海量无标注文本上学习语言规律
- **微调**：在特定任务数据上调整模型参数
- **指令微调**：让模型学会遵循指令

### 3. 缩放定律（Scaling Laws）

模型性能与参数量、数据量、计算量呈幂律关系，越大越强。当模型规模达到一定阈值后，会突然展现出在小规模模型中完全不存在或表现不佳的全新能力，即**能力涌现**。

## 📝 自回归生成过程

GPT 的工作模式被称为**自回归（Autoregressive）**：

1. 给模型一个起始文本（例如 "Datawhale Agent is"）
2. 模型预测出下一个最有可能的词（例如 "a"）
3. 模型将自己刚刚生成的词 "a" 添加到输入文本的末尾，形成新的输入
4. 模型基于这个新输入，再次预测下一个词（例如 "powerful"）
5. 不断重复这个过程，直到生成完整的句子或达到停止条件

## 💡 实例/案例

**GPT 的生成过程**：

```
输入："今天天气很好，我想去"

第1步：预测下一个词 → "公园"
输入更新："今天天气很好，我想去公园"

第2步：预测下一个词 → "散步"
输入更新："今天天气很好，我想去公园散步"

第3步：预测下一个词 → "。"
输出完成："今天天气很好，我想去公园散步。"
```

## 📝 个人笔记

GPT 的成功证明了一个简单范式的威力：预测下一个词。这个看似简单的目标，在海量数据和超大模型的加持下，涌现出了惊人的能力。在智能体开发中，GPT 系列模型是目前最可靠的基座模型选择。
