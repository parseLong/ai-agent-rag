# Manus

> Monica于2025年3月6日推出的通用型AI Agent，采用多智能体协同架构+CodeAct执行引擎+外部化记忆，从"对话交互"跃迁到"人机协作"。名字取自拉丁语"手"（Mens et Manus——手脑并用），象征从思维到行动的桥梁。

## 核心要点

- **技术路线**：选择==上下文工程==（Context Engineering）而非端到端自训练模型，产品迭代周期从数周缩短到数小时
- **多智能体架构**：规划(Planner)-执行(Execution)-验证(Verification)三层闭环
- **CodeAct执行引擎**："代码即行动"，将所有Agent行动转化为可执行Python代码
- **外部化记忆**：文件系统作为"无限上下文"，上下文长度缩短80%+且无信息丢失
- **异步处理**：用户关闭设备后任务仍持续运行，Checkpointing机制每15分钟保存状态

> [!important] 核心创新
> Manus的核心贡献不在底层模型突破，而在==上下文工程编排能力==——动态调度多模型资源（Claude/DeepSeek等），集成RL算法+工具包生态。实现了从"生成答案"到"==执行闭环=="的跃迁：思考→规划→执行→验证→交付成果。

## 一、技术路线抉择：上下文工程 vs 端到端模型

| 维度 | 端到端自训练模型 | 上下文工程（Manus路线） |
|------|----------------|----------------------|
| 迭代周期 | 数周-数月 | 数小时-数天 |
| 模型依赖 | 强依赖，模型淘汰则产品失效 | 弱依赖，随模型进步而提升 |
| 灵活性 | 低，难以适配新任务 | 高，无缝切换模型生态 |

**押注上下文工程的四大理由**：
1. **历史教训**：团队吸取前一轮创业"自研模型被淘汰"的惨痛经历，放弃重资产路线
2. **工程效率**：产品迭代周期从数周缩短到数小时
3. **模型解耦**：作为"船"随潮水而动，底层模型进步时自动受益
4. **生态适配**：支持自托管模型与API调用无缝切换

> 上下文工程本质：**"手工随机梯度下降"**——团队短时间内四次重构Agent框架，通过架构搜索→提示调整→性能评估的循环，不断逼近局部最优解。

## 二、分布式多智能体架构（PEV架构）

Manus采用**规划-执行-验证三层模块化架构**，实现任务全闭环管理：

### 2.1 三大核心Agent

| Agent类型            | 核心职责                         | 技术实现                   | 关键指标                       |
| ------------------ | ---------------------------- | ---------------------- | -------------------------- |
| **规划代理(Planner)**  | 解析意图、拆解子任务、动态优先级调度、生成todo.md | 强化学习优化任务分解策略+思维链推理     | 子任务拆解准确率>92%，优先级调整响应<500ms |
| **执行代理(Executor)** | 调用工具/代码/API完成操作、环境交互         | CodeAct机制+Docker沙盒执行环境 | 工具调用成功率>90%，执行延迟<2s        |
| **验证代理(Verifier)** | 结果交叉校验、逻辑验证、错误检测、迭代优化        | 多轮测试+反馈闭环+模型自评估        | 错误识别率>85%，结果修正率>80%        |

### 2.2 Validation Loop相互校验系统

- **状态同步**：各Agent通过共享内存实时同步任务进度与执行结果
- **错误传递**：执行失败时自动触发验证代理介入，生成修正方案
- **动态调整**：规划代理根据验证结果实时调整子任务优先级与执行路径
- **鲁棒性提升**：系统整体容错能力提升约60%

### 2.3 运行环境

- 基于云的沙盒（cloud-based sandbox），每个任务创建独立"数字工作空间"
- 本质上是基于浏览器的智能体，在Linux沙盒中操作
- Docker Ubuntu 22.04 LXC容器，每个任务独立
- 资源限制：CPU≤2核，内存≤4GB，网络带宽≤10Mbps

## 三、CodeAct机制："代码即行动"的执行引擎

> [!tip] 核心创新
> 区别于传统"工具调用+参数传递"模式，CodeAct将所有Agent行动统一转化为可执行的Python/JavaScript代码。

### 3.1 16种核心Action空间

| 操作类型             | 准确率  | 适用场景         |
| ---------------- | ---- | ------------ |
| Computer Use     | 95%+ | 桌面应用自动化      |
| Browser Use      | 92%+ | 网页数据爬取、自动化测试 |
| Shell Command    | 98%+ | 系统配置、环境搭建    |
| Code Execution   | 90%+ | 数据分析、模型训练    |
| API Call         | 88%+ | 第三方服务集成      |

| PDF Processing   | 93%+ | 文档分析、报告生成    |
| Image Processing | 91%+ | 图像分析、内容提取    |

### 3.2 自纠错重试机制

- 执行失败自动分析错误类型（语法/运行时/环境错误）
- 最多重试3次，失败重试率85%以上

- "视觉-行动"闭环：截图→分析→生成代码→执行→验证

- 代码片段复用 + 资源动态调度

## 四、KV缓存：性能生命线

- 平均输入输出token比例：**100:1**（复杂任务可达1000:1）
- 推理成本降低**10倍以上**（Claude Sonnet：缓存输入0.3美元/百万token vs 未缓存3美元/百万token）

### 提高KV缓存命中率的五大准则

1. **稳定提示前缀**：避免在系统提示中引入时间戳等动态内容
2. **上下文仅追加不修改**：确保序列化过程确定性，避免JSON键顺序变化
3. **显式标记缓存断点**：手动插入断点，合理分配缓存空间
4. **分布式一致性路由**：通过会话ID确保跨工作器的缓存一致性
5. **工具定义固化**：保持工具描述稳定，避免频繁修改

> 缓存失效代价：推理延迟增加3-5倍，成本上升10倍。应对：缓存预热机制 + 缓存失效预警系统

## 六、外部化记忆：文件系统作为"无限上下文"

### 创新方案

- 将文件系统视为"无限上下文"，模型可按需读写文件
- **可逆压缩策略**：保留URL即可删除网页内容，保留路径即可省略文档内容
- 上下文长度缩短80%+，同时不丢失关键信息

### 实践应用

- **todo.md动态更新**：不断重写该文件，将全局计划复述到上下文末尾
- **错误日志持久化**：保存失败操作与观察结果
- **多Agent记忆共享**：通过文件系统实现跨Agent状态同步

## 七、分层记忆管理：Working-Hot-Cold三层体系

| 记忆层级               | 存储内容         | 生命周期  | 技术选型                  | 容量限制          |
| ------------------ | ------------ | ----- | --------------------- | ------------- |
| **Working Memory** | 当前任务上下文、执行步骤 | 任务存续期 | 大模型上下文窗口+本地缓存         | 8k-32k tokens |
| **Hot Memory**     | 近期任务经验、用户偏好  | 7-30天 | Redis+向量索引            | 1M+ tokens    |
| **Cold Memory**    | 领域知识库、历史任务归档 | 永久    | Chroma/Milvus向量库+知识图谱 | 无限（文件系统）      |

**关键优化**：
- **自编辑记忆**：Agent可自主更新记忆内容
- **LangGraph Store优化**：检索响应时间<100ms
- **记忆优先级调度**：动态调整各层级记忆访问权重

## 八、工具管理：屏蔽而非移除的动态状态机

- Manus支持**47种原生工具**+用户自定义工具
- 工具数量爆炸导致模型选择错误概率增加、缓存失效
	- **缓存噩梦**：如果为了减少干扰而动态删减工具列表（例如这轮只给浏览器工具，下轮只给代码工具），那么**系统提示词/工具定义部分每次都在变**。根据KV缓存原理，前缀一变，整个缓存全部作废，每轮对话都要重新计算数千Token的工具定义。
- **解决方案**：不直接删除工具，在解码期间通过**屏蔽token logits**约束动作选择
- 设计一致前缀的动作（如`browser_`、`shell_`），便于分组屏蔽
- 响应预填充机制：自动/必需/禁止三种函数调用模式
- **极大正面影响KV缓存命中率**

## 九、注意力操纵：减少任务漂移

### 三大策略

1. **todo.md全局计划更新**：每5-10步操作后重新生成
2. **上下文末尾追加摘要**：每次工具调用后自动生成任务进度摘要
3. **自然语言强化提示**："现在我需要完成...""我的最终目标是..."

**效果**：多步任务完成率提升40%+，任务漂移概率降低65%

## 十一、安全机制

### 11.1 gVisor沙箱隔离

- 基于Google gVisor容器技术，任务进程与宿主机资源隔离
- 成功防御99.6%的恶意指令攻击

### 11.2 零信任安全防御体系

| 防御层 | 机制 | 说明 |
|-------|------|------|
| **输入验证层** | 语义合法性检查+权限校验+敏感信息脱敏 | 过滤异常指令，屏蔽API密钥等凭证 |
| **执行监控层** | 行为基线建模+异常检测+动态策略调整 | 实时分析系统调用，触发限流或熔断 |
| **审计追溯层** | 全链路日志+操作回滚+攻击模式学习 | 保存完整执行轨迹，支持事后撤销 |

### 11.3 数据隐私保护

- **混合架构**：LLM + 本地知识库，客户数据泄露风险降低87%
- **三重加密**：传输层TLS 1.3 / 存储层AES-256 / 访问控制RBAC
- **FHE全同态加密**：数据"可用不可见"
- 通过ISO/IEC 27001认证，支持私有化/本地化部署
