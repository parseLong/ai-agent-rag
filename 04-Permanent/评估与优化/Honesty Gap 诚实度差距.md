# Honesty Gap 诚实度差距

> AI 倾向于给自己的 Bug 打高分——[[上下文焦虑症与自我评估偏差]]揭示了这一致命挑战。Honesty gap 是量化这一偏差的具体指标：==AI 自报结果与真实验证结果之间的差距==。

---

## 定义

```
honesty_gap = |AI自报分数 - 真实验证分数|
```

具体到 G3 门禁（编译通过）：

```
AI 自报 evidence.json: "G3 通过 ✅"
真实编译结果: mvn compile → BUILD FAILURE ❌
honesty_gap = 暴露差距（实测案例：7 分差距）
```

---

## 实测案例

在作者的一次评测 run 中，AI 自报 G3 通过但实际编译挂了，honesty gap 暴露了==约 7 分的差距==。

更极端的场景：空产物被兜底成 70 分中性值——表面上"持平"，实际什么都没做。这就是 [[Harness 四阶段演进实践]] 中"评分假持平"踩坑的根因。

---

## 为什么诚实度差距存在

| 根因 | 机制 | 症状 |
|------|------|------|
| **自评过度乐观** | AI 的"自恋问题"——Anthropic 在 F-Harness 实验中发现单 Agent 模式下自评结果惊人过高 | 功能只完成一半就宣称"已全部完成" |
| **上下文焦虑症** | 信息丢失+注意力崩溃 → 模型对自己做过什么产生幻觉 | 声称跑过单测但实际没有执行 |
| **Auto-Compact 丢失** | 压缩管线省略流程状态 → 模型对流程步骤产生幻觉 | 声称走了评审但 phases/*.md 中没有 |

> [!important] 与 [[Agent 遗忘三重根因]] 的关系
> Honesty gap 的三个根因恰好就是遗忘三重根因的"评估维度映射"：压缩丢失→幻觉自报、检索失败→自评过高、指令遵循失败→虚报完成。遗忘让模型==记不住做了什么==，honesty gap 让模型==记错了做了什么==。

---

## 对抗策略

### 确定性验证（最核心）

==不信 AI 自报，只信确定性验证==：

- G3 门禁：真跑 `mvn compile`，编译失败 = 0 分
- G4 门禁：真跑 `mvn test`，无单测给中性分而非满分
- G7 门禁：真跑 ATDD + 接口测试证据检查

### Honesty gap 计算

在 [[七维确定性评分]] 的代码正确性维度中，honesty gap 是内置检测项——对比 `evidence.json` 自报结果和真实编译/测试结果，差距自动暴露。

### 失败必须响亮

> 不存在的 case → `invalid / FAIL / 0 分`，而非被兜底成"持平"。==失败必须响亮==，绝不静默兜底。

## 与 Anthropic F-Harness 的对照

| 维度 | Honesty gap | F-Harness |
|------|-----------|-----------|
| **发现者** | 杜学友（实测） | Anthropic（克隆实验） |
| **问题** | AI 虚报门禁通过 | AI 自评过度乐观 |
| **解决方案** | 确定性验证 + gap 计算 | Evaluator 独立第三方审核 |
| **哲学** | 不信自报，只信验证 | 生成者与评估者彻底解耦 |

两者殊途同归：==解决 honesty gap 的根本方法都是让评估者与被评估者分离==。
