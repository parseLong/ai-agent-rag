# PEAS 模型

## 📖 定义

PEAS 是描述智能体任务环境的经典模型，包含四个要素：

- **P**erformance（性能度量）
- **E**nvironment（环境）
- **A**ctuators（执行器）
- **S**ensors（传感器）

## 🧠 我的理解

PEAS 就像给智能体画一张"身份证"，说明它在什么环境下工作、用什么感知、用什么行动、怎么算成功。

## 🎯 为什么重要

设计 Agent 时，首先要明确它的任务环境。PEAS 模型帮助系统性地定义这些要素。

- **应用场景**：[[智能旅行助手]]、[[自动驾驶]]

## 💡 实例/案例

**智能旅行助手的 PEAS 描述**：

| 要素 | 描述 |
|------|------|
| **P**erformance | 用户满意度、行程合理性、预订成功率 |
| **E**nvironment | 互联网、航班/酒店 API、天气数据 |
| **A**ctuators | API 调用、数据库查询、生成回复 |
| **S**ensors | 用户输入、API 返回数据、网页内容 |

## 📝 个人笔记

PEAS 不仅是一个描述工具，更是一种思考方式。在定义任务环境时，要注意环境的几个关键特性：
- **部分可观察性**：Agent 无法获取全部信息
- **随机性**：行动结果不确定
- **动态性**：环境状态会变化
- **多智能体**：环境中存在其他智能体
