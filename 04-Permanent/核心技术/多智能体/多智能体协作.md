# 多智能体协作

> 多个专用Agent协作完成复杂任务，比一个大而全的Agent效果更好。

## 核心要点

- **三大优势**：专业化分工（每个Agent针对特定任务优化）、模块化易维护（独立开发部署）、并行处理（缩短执行时间）
- **三大挑战**：通信协调（Agent间怎么交换信息）、任务分配（复杂任务怎么拆解给谁）、冲突解决（多个Agent结论不一致时怎么仲裁）
- **协作架构**：
  - 监督者模式：一个Supervisor协调多个Worker
  - 流水线模式：按顺序处理，前一个输出作为后一个输入
  - 多轮协商模式：反复讨论直到达成共识

> [!note] 更详细的7种协作模式
> 以上3种架构是多智能体协作的基础模式。生产环境中还存在更多细分模式，详见 [[多智能体协作模式]] ——涵盖主从委托、接力传递、对等讨论、层级分治、竞争选优、评估反馈、动态路由7种核心模式及其组合策略。

> [!warning] 多Agent常见失败原因
> - 规范问题：不遵循任务要求、步骤重复、不知何时终止
> - 沟通问题：误解、信息不流通、各行其是
> - 任务验证问题：没检查或检查没查出问题

## 多智能体架构的演进视角

> 来源：[[Agent技术范式演变（2023-2026）]]

多智能体协作的形态随着 Agent Planning 能力的增强而发生了根本变化——从"人工编排"到"Agent 自主动态构建"：

| 阶段 | 协作形态 | 特征 |
|------|---------|------|
| **早期 Agent** | 人工预定义多Agent分工 | 如写作Agent+编辑Agent+绘图Agent，每个Agent独立 System Prompt |
| **工作流 Agent** | 固定 Workflow 编排 | Workflow 引擎驱动步骤跳转，LLM 作为节点嵌入 |
| **自主 Agent** | 子Agent动态构建 | 根据子任务需求动态实例化专项子Agent，从"单体思考"到"协同作战" |

> [!important] 关键转变
> 从"人工定义每个Agent的角色和Prompt"到"Agent自主根据任务需要动态构建子Agent"——这标志着多智能体协作从"工程编排"走向"智能协同"。详见 [[任务分解]] 中的"子Agent动态构建"概念。
