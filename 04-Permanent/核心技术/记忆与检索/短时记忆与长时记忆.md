# 短时记忆与长时记忆

> Agent 记忆有**时间轴**（Working vs Long-term）和**内容轴**（Episodic vs Semantic vs Procedural）两个维度，两者**正交、不互斥**。

## 🔑 时间轴：Working vs Long-term

| 比较维度 | [[短时记忆]] / 工作记忆 | [[长时记忆]] / 持久记忆 |
|---|---|---|
| **核心意思** | 当前 session 中模型**当下看得到**的信息 | 存在外部、之后可**跨 session 取回**的信息 |
| **持续时间** | 短，通常限于当前 session | 长，可跨 session |
| **技术基础** | [[上下文窗口]] / prompt | [[记忆系统]] 存储层 / [[向量存储]] |
| **适合记什么** | 任务细节、刚刚说过的内容 | 用户偏好、长期目标、背景资料 |
| **是否受 context 长度限制** | **会**，模型一次能看到的内容有限 | 较不受限，需要时只取一小段放回上下文 |

> [!note] 术语提醒
> 在 agent 里，"短期记忆"更准确的说法是 **working memory**。它不是外部存储，而是当前 prompt / [[上下文窗口]] 里看得到的内容。

## 🔑 内容轴：三种记忆类型

| 类型 | 中文 | 核心意思 | 范例 |
|---|---|---|---|
| **Episodic memory** | 情节记忆 / 经验记忆 | 某次任务/互动/失败的具体经验 | "上次做这个任务失败了、原因是..." |
| **Semantic memory** | 语义记忆 / 事实记忆 | 稳定知识、用户偏好、背景事实 | "用户是素食者"、"Python 用 4 空格缩进" |
| **Procedural memory** | 序记忆 / 技能记忆 | agent 知道"怎么做事"的规则 | [[记忆到技能转化]] — 如何调用某个 tool |

> 两组分类**正交**：[[长时记忆]]里可以同时有三种。Reflexion 是典型的 episodic memory 应用。

## 🔑 CoALA 框架 — 4 层分类法

[Sumers et al. 2023](https://arxiv.org/abs/2309.02427) 把 agent memory 分成 4 种：

| 类型 | 存储什么 | 对应示例 |
|---|---|---|
| **Working memory** | 当前任务上下文 | LLM [[上下文窗口]] 本身 |
| **Episodic memory** | 过去任务的具体经验 | Reflexion 记录、过往 trajectories |
| **Semantic memory** | 抽象事实 / 知识 | [[RAG]] 知识库、用户画像 |
| **Procedural memory** | 如何执行动作 / 技能 | [[工具调用（Function Calling）]] definitions |

> CoALA 可以当作**检查表**，帮你看出 agent 缺了哪一层。Production agent 往往需要兼顾全部 4 层。

## 🔑 3 种设计 Pattern

| Pattern | 适合场景 | 怎么跑 | 成本 |
|---|---|---|---|
| **1. Naive buffer** | 短对话 ≤10 turn、不跨 session | 全段 history 送进 prompt | [[Token]] 线性增长 |
| **2. Summary + recent** | 中长对话 ~50 turn | 每N轮LLM摘要旧history；prompt=summary+last N | 中等 |
| **3. [[向量存储]] + retrieval** | 跨 session + 知识库 | embed过去消息→[[向量存储]]→每轮query相关片段 | 高但[[Token]]稳定 |

**怎么选**：
- 对话 chatbot 无跨 session → **Pattern 1**
- agent 长对话需记住今天聊过什么 → **Pattern 2**
- agent 跨 session + 知识库 → **Pattern 3**
- Production 大型 agent → **混用** Pattern 2 + 3

## 🔑 Generative Agents — 三重评分加权（经典参考）

[Park et al. 2023](https://arxiv.org/abs/2304.03442) 的 Smallville 模拟使用三个分数加权：

- **Importance**：LLM 打1-10分（吃饭=2分、分手=9分）
- **Recency**：时间指数衰减
- **Relevance**：与当前 query 的 embedding 相似度

最终得分 = `α·importance + β·recency + γ·relevance`

> 这是 mem0 / Letta 等生产 memory layer 的**概念骨架**。
