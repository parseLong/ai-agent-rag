# 向量存储

> 将文本等非结构化数据转换成固定维度的数值向量，存储在专门的数据库中支持高效的相似度搜索。

## 核心要点

- **选型考虑**：规模、性能、成本三方面权衡

| 数据库           | 特点              | 适用场景     |
| ------------- | --------------- | -------- |
| Faiss         | Facebook开源，本地部署 | 小规模本地    |
| [[Milvus]]    | 分布式，混合查询        | 大规模生产    |
| Pinecone      | 托管服务，开箱即用       | 快速验证     |
| Weaviate      | 支持语义搜索          | 功能丰富场景   |
| pgvector      | PG扩展            | 已有PG基础设施 |
| Elasticsearch | 支持混合搜索          | 关键词+向量场景 |

- **索引算法**：HNSW基于图结构（构建慢查询快，适合静态数据）；IVF基于倒排文件（构建快查询慢，适合动态更新）
- **相似度度量**：余弦相似度（最常用，关注方向不关注长度）、欧氏距离（直观简单）、点积（计算最快）
