# 层次化注意力机制

> 无限画布的目的不是把上下文变大，而是让 Agent 的注意力变得更有层次。Agent 不需要记住所有内容，只需要知道哪些信息重要、它们被组织在哪里、必要时如何一步步展开。

## 三层注意力模型

有了 [[Mermaid任务画布]] 之后，Agent 的信息处理方式从扁平变为层次化——像人类处理复杂资料一样，分层查看信息：

### 第一层：鸟瞰（Overview）

Agent 先看到的是**任务级别的概览**：之前做过哪些任务、哪些已经完成、哪些还在进行、当前用户的问题可能和哪个历史任务有关。

这一层不追求细节，而是帮助 Agent **快速判断方向**——对应折叠态的 Metadata。

### 第二层：聚焦（Focus）

当 Agent 判断某个任务相关时，它打开对应的任务画布，查看**任务内部的结构**：哪些步骤已经完成、哪些节点还在进行、哪些信息汇聚成了当前结论。

这一层让 Agent 看到经过组织的**任务地图**——对应展开态的 Active MMD。

### 第三层：下钻（Detail）

当画布节点信息仍然不够时，Agent 才追溯到更细的摘要或原始材料。原文不是一直常驻上下文，而是在**真正需要确认细节时才被重新展开**——对应 JSONL 记录和 refs 原文。

```
Overview → Metadata（任务级索引）
  ↓ 打开相关任务
Focus → MMD 画布（任务步骤级摘要）
  ↓ 需要细节时
Detail → JSONL → refs/*.md（工具调用级摘要 → 原始证据）
```

## 类比：写报告的认知过程

层次化注意力很像我们自己写报告的认知过程：

| 层次 | 人类行为 | Agent 行为 |
|------|---------|-----------|
| Overview | 先看目录，知道有哪些章节 | 先看 Metadata，知道有哪些历史任务 |
| Focus | 打开相关章节，看结构和重点 | 打开 MMD 画布，看任务结构和进度 |
| Detail | 需要核对细节时，翻原始文献 | 需要确认细节时，读 JSONL 和 refs 原文 |

我们不会一上来就翻所有原始资料，而是**逐层下钻**——只在真正需要时才展开最细的层级。

## 避免两种极端

层次化注意力机制避免了两种极端：

| 极端 | 问题 | 层次化注意力的应对 |
|------|------|------------------|
| **所有信息塞进上下文** | Token 浪费 + 注意力稀释 | 信息分层，当前只保留 Overview/Focus 层 |
| **过度摘要** | 省了 Token，但原始信息无法恢复 | Detail 层保留在外部，可按需下钻找回 |

> [!important] 核心洞见
> **压缩不是让 Agent 少知道，而是让 Agent 少背负；信息可以离开上下文窗口，但不能离开 Agent 的可达范围。**

## 与 [[上下文卸载]] + [[Mermaid任务画布]] 的对应关系

层次化注意力是卸载 + 画布组合方案的**使用方式**——决定了 Agent 如何在四级折叠架构中导航：

| 层次 | 对应存储层 | 信息粒度 | 常驻上下文？ |
|------|-----------|---------|------------|
| Overview | Metadata (Level 3) | 任务级索引 | ✅ 历史任务入口保留 |
| Focus | MMD Node (Level 2) | 任务步骤级摘要 | ✅ 当前任务 Active MMD |
| Detail-L1 | JSONL (Level 1) | 工具调用级摘要 | ❌ 按需从外部文件读取 |
| Detail-L2 | refs/*.md (Level 0) | 原始证据 | ❌ 按需从外部文件读取 |
